Breadcrumb

Shows where the user is in the product and lets them step back up the hierarchy. A Back action leads, followed by the trail of pages.

Anatomy

A full-width bar with a Back action, the page trail, and an optional action on the far end.

  1. Projects
  2. Company pulse
  3. Reporting structures
1
Back — A secondary button that takes the user up one level. It leads the bar so the most common action is always in the same place.
2
Crumb — A link to an ancestor page. Selecting it jumps straight to that level.
3
Separator — A slash between crumbs, purely decorative.
4
Current page — The last crumb, marked as the current location and not clickable.
5
Action — An optional button on the far end for a page-level action (e.g. Export). Leave it out when the page has no such action.

Usage

Use a breadcrumb to orient users inside a deep hierarchy and let them move back up — not as the main way to get around the product.

Orient, don't replace navigation

  1. Surveys
  2. Q2 engagement
Do
Show the path to the current page so users know where they are and can step back up.
  1. Home
  2. Surveys
  3. Reports
  4. Settings
  5. Team
Don't
Use it as the primary navigation. The trail reflects hierarchy — it isn't a menu of everywhere a user can go.

Keep it on one line

/ Reporting structures
Do
When space is tight, collapse the middle of the trail into a “…” that opens the full path.

  3. Reporting structures
  4. Coordinator view
Don't
Let the trail wrap onto a second line. Collapse it instead.

Variants

The trail grows with depth, can carry a trailing action, and collapses on small screens.

  1. Surveys

Single level — One step down from the top. The Back button plus the current page is enough; there's no trail yet.

  1. Projects
  2. Company pulse
  3. Reporting structures

Multi-level — Deeper pages show the full trail of ancestors so any level is one click away.

  1. Projects
  2. Reporting structures

With trailing action — A page-level action sits on the far end, balanced against the trail. Use it only for an action that belongs to the whole page.

/ Reporting structures

Collapsed (mobile) — On narrow screens the Back button becomes an icon and the middle of the trail folds into a “…”. Selecting it reveals the full path in a “Navigate to” menu (below).

Content guidelines

Crumbs mirror page titles — short, in the same words the user clicked to get there.

Match the page title

Each crumb should read exactly like the title of the page it links to, so the trail feels predictable.

  1. Surveys
  2. Q2 engagement
Do
Real page names, in sentence case.
  1. Section 2
  2. DETAIL VIEW / SUB-PAGE
Don't
Generic or shouty labels that don't match any page title.

Keep crumbs short

Trim long titles to a recognisable few words so the trail stays on one line.

  1. Reports
  2. Reporting structures
Do
A couple of clear words per crumb.
  1. Reports
  2. Reporting structures for the whole organisation, 2026
Don't
Full sentences that overflow and force the trail to wrap.

Accessibility

A breadcrumb is a navigation landmark with an ordered trail and a clearly marked current page.

→
Wrap it in a nav landmark. Use <nav aria-label="Breadcrumb"> so assistive tech announces and can skip to it; the label distinguishes it from other navs.
→
Use an ordered list. The trail is an <ol> — the order carries meaning (top level → current page).
→
Mark the current page. Give the last crumb aria-current="page" and render it as text, not a link.
→
Hide the separators. Slashes are decorative — draw them with CSS or mark them aria-hidden so they aren't read out.
→
Label icon-only controls. The collapsed Back and “…” buttons need an aria-label (“Back”, “Show full path”), and the “…” opens a menu with the full, reachable trail.
→
Keyboard reachable. Back, every crumb and the “…” menu are tab-stops with a visible focus ring.
