Card

A surface that groups related content and actions into a single, scannable block. The base building block for dashboards, lists and overviews.

Anatomy

A rounded surface with optional media, a body (title + text) and an actions row.

Card title Whatever the card needs to hold — text, metrics, a chart, a list.
1
Surface — The rounded container with the chosen variant (border/shadow/fill).
2
Body — The padded content area. The content itself is fully dynamic — text, metrics, a chart, a list, form fields, anything.
3
Actions — Optional row of buttons or links at the bottom.

Usage

Use a card to group content that belongs together and benefits from a clear boundary. Don't wrap every element in a card — too many borders create visual noise.

Variants

Three surface styles, in increasing quietness. Pick one per surface and stay consistent within a view.

ElevatedRaised on a soft shadow.
Elevated — A hairline border and a soft shadow lift the card off the page (the styleguide default). Use for standalone or clickable cards that should stand out.
OutlinedA quiet 1px border, no shadow.
Outlined — A thin border, no shadow. The calm default — works well in grids and dense layouts where many shadows would be noisy.
FilledA tinted surface, no border.
Filled — A subtle grey fill instead of a border. For secondary or nested surfaces that sit on a white background.

Density

Padding scales with the card's size — tighter for small cards, roomier for large ones.

Default — 24pxLarge cards and standalone surfaces.
Default — 24px padding. The standard for large or standalone cards.
Small — 16pxSmaller cards in a grid.
Small (.is-sm) — 16px padding for smaller cards, e.g. tiles in a dense grid.
Compact — 12pxTight, content-dense cards.
Compact (.is-compact) — 12px padding for the tightest, most content-dense cards.

Accessibility

A card is a container; make any interactivity explicit.

→
Clickable cards. If the whole card is a link, wrap it in a single <a> (or <button>) so it's one focusable, announced target — don't scatter nested links that duplicate it.
→
One primary action. Keep the actions row to one primary action plus, at most, a secondary one.
→
Heading structure. The card title should use a real heading level that fits the page outline, not just bold text.
→
Don't rely on the surface alone. Elevation/border groups content visually; the content order must still make sense linearly.

Production card (Angular styleguide)

The Angular styleguide ships a single .card (border + shadow, 8px radius) built from a .card-title-container (with a bottom divider) and a .card-content-container — both 24px padding. The title is an <h2 class="card-title text-l5">.

<div class="card">
  <div class="card-title-container">
    <h2 class="card-title text-l5">Card title</h2>
  </div>
  <div class="card-content-container"> … </div>
</div>
⚠️
Figma / styleguide gap The Angular styleguide currently has one card (the production markup above: border + shadow, a title-container with a divider + a content-container) plus a with-border flag — there's no elevated / outlined / filled split. The three variants on this page are a proposed Material/Atlassian-style extension; reconcile the variant names, the title-divider and the slot names (card-body vs card-content-container) with the styleguide and Figma before standardising.

Accessibility

Keep interactivity and headings honest.

→
One target. A clickable card is a single <a>/<button>, not a div with a click handler.
→
Heading. .card-title should be a real heading element where it fits the outline.
→
Focus visible. Interactive cards need a clear focus ring (the link/button provides it).
