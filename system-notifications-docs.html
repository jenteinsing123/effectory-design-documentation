System Notifications

A dark, system-level message that reports the outcome of a background process or a system event. It appears top-right of the screen with an optional action and dismiss.

Anatomy

A dark card with a coloured accent bar, a title, an optional description and action, and a dismiss control.

Notification message
Write a description here if the notification needs it
1
Accent bar — A coloured edge that signals the sentiment (success, error, warning).
2
Title — A short summary of what happened. Always present.
3
Description — Optional supporting line with more detail.
4
Action — Optional single link to follow up (e.g. View, Retry, Undo).
5
Dismiss — Closes the notification.

Usage

Use a system notification for the result of a system or background event — a save that finished, a sync that failed, a session about to expire. For feedback tied to a specific element on the page, use an Inline notification instead.

One clear action

Export ready
Your report finished exporting.
Do
One action at most, naming the follow-up.
Something went wrong while saving your changes to the reporting structure and we could not recover them automatically
Don't
Cram a paragraph into the title — keep it short, use the description.

Variants

The sentiment sets the accent colour and the meaning. The title, description and action are otherwise identical.

Placement & motion

System notifications always appear in the top-right of the screen, 8px from the edges, and stack downwards when there are several. New ones slide up into place while fading in.

Export ready
Your report finished exporting.

Fixed top-right placement. The enter animation is a short slide-up + fade (≈200ms).

Content guidelines

Notifications are read in a glance — lead with the outcome.

Title states the outcome

Say what happened in the title; put the detail in the description.

Report exported
Saved to your downloads.
Do
Outcome first, detail below.
Notification
Please be aware that the thing you did has now been done successfully.
Don't
Vague title + wordy description.

Accessibility

Notifications must be announced and dismissable.

→
Live region. Success uses role="status" (polite); Error and Warning use role="alert" (assertive) so screen readers announce them.
→
Dismiss label. The close button needs an accessible label (aria-label="Dismiss") — the icon alone isn't enough.
→
Don't rely on colour alone. The accent signals sentiment, but the title text must carry the meaning too.
→
Give time to read. If it auto-dismisses, allow enough time and pause on hover/focus; errors should persist until dismissed.

Usage

A dark card with a sentiment modifier, built from the .sysnotif classes.

Success

Export ready
Your report finished exporting.
<div class="sysnotif" role="status">
  <div class="sysnotif-title">Export ready</div>
  <div class="sysnotif-desc">Your report finished exporting.</div>
  <button class="sysnotif-action">Download</button>
  <button class="sysnotif-close" aria-label="Dismiss"><i data-icon="cross"></i></button>
</div>

Error & Warning

Add is-error or is-warning and use role="alert".

Sync failed
We couldn't reach the server.
Session expiring soon
You'll be signed out in 5 minutes.
<div class="sysnotif is-error" role="alert"> … </div>
<div class="sysnotif is-warning" role="alert"> … </div>

Placement

Mount notifications inside a .sysnotif-stack — a fixed, top-right container that stacks them and lets clicks pass through the empty area. Each notification slides up and fades in (≈200ms).

<div class="sysnotif-stack">
  <div class="sysnotif" role="status"> … </div>
  <div class="sysnotif is-error" role="alert"> … </div>
</div>
⚠️
Confirm the Angular component The markup above is the design-system HTML/CSS structure. Check the Angular styleguide (styleguide.effectory.com) for a dedicated notification service/selector before building — if one exists, use it instead of hand-rolling the markup.
