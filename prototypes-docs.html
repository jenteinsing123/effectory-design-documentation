Get started

Build realistic product screens that look exactly like the real app — by describing what you want in plain language, directly in Claude Code.

What is it

A way to build product screens with the real components and styles of this design system — no Figma, no coding required.

The Engage Design System ships as a skill for Claude. When you describe a screen — in plain Dutch or English — Claude builds a working HTML prototype using the exact same components, colours, and spacing as the real product. The result looks like the real app because it uses the real app's styles.

You don't need to know how to code. You just need Claude open and a rough idea of what you want to build.

For Effectory designers

Nothing to install — the skill is already in your Claude workspace.

✅
You're good to go. Every Effectory teammate signed in to the company Claude account automatically has the effectory-design-system skill available. No download, no terminal commands.
1
Open Claude with your Effectory account
Sign in to Claude using your Effectory login. You can use Claude in the browser, the desktop app, or Claude Code — the skill is available in all of them.
2
Describe what you want to build
Just type your request. Use words like prototype, design, scherm, pagina, or mockup and Claude will recognise it and load the skill automatically. Examples:
"bouw een prototype van het instellingenscherm" "maak een design voor een onboarding-flow met drie stappen" "prototype van een dashboard met filters en een tabel"
3
Claude builds the screen and shows you the result
Claude generates an HTML preview directly in the conversation. You can iterate by asking for tweaks — "make the button primary", "add a back link", "use dark mode" — and Claude updates the file in place.

For everyone else

If you're not part of Effectory, you can still use the skill by installing it from GitHub.

1
Download the skill from GitHub
The skill is open source and lives in our public GitHub repo. You don't need a GitHub account to download it.

Download design system (.zip)
Or browse the source at github.com/jenteinsing123/effectory-design-documentation. Unzip and save the folder somewhere easy to find.
2
Install Claude Code
Claude Code is a free CLI tool by Anthropic. Open Terminal and run:
npm install -g @anthropic-ai/claude-code

If you see npm not found, install Node.js first — it ships with npm.
3
Open Claude Code inside the folder
In Terminal, navigate to the folder you downloaded in step 1 and run:
claude

Claude Code starts and automatically picks up the effectory-design-system skill from the .claude/skills/ folder. From here, describe what you want to build — exactly like the Effectory team path above.
🔄
Keep it up to date. The design system gets new components and improvements regularly. Re-download the ZIP before each session, or — if you cloned with Git — run git pull in the project folder.

Viewing your prototype

Open it through the local server — not by double-clicking the file.

⚠️
Don't open the file by double-clicking it. This breaks icons and some visual details — the prototype will look incomplete even though nothing is wrong. Always use the local server instead (see below).
1
Start the local server
In a Terminal window inside the project folder, run:
python3 serve.py

This starts a small local web server on your computer. Leave this Terminal window open while you're working.
2
Open the link Claude gives you
After building, Claude always prints a URL like http://localhost:8000/prototypes/my-screen.html. Copy that link and paste it into your browser.

How it works

Claude follows a few strict rules that keep prototypes consistent with the real product.

Only real components. Claude only uses components, colours, and icons that actually exist in the design system. It never invents something new or uses a rough approximation.
If something doesn't exist, Claude tells you. If you ask for a component that isn't in the system yet, Claude stops and tells you clearly — it won't silently build a substitute that looks different from the real product.
Prototypes look like the real product. Not a documentation page, not a wireframe — a realistic product screen. Dark mode and portal theming (Coordinator / Personal) work out of the box.
Files are saved automatically. Every prototype is saved in the prototypes/ folder inside the project. You can share the URL with teammates — as long as they're also running the local server.

For design system maintainers

Three files need to stay in sync with the documentation whenever components or tokens change.

File Update when
components.css A component is added or its CSS changes. Bundle the updated <style> block from the component's doc page into this file.
foundation.css Spacing, radius, shadow, or typography variables change. Mirrors the :root values from styles.css that prototypes need.
.claude/skills/effectory-design-system/design-system-reference.md A new component, variant, token, or icon is added. This is the reference Claude reads before building — if it's not listed here, Claude won't use it.
tokens.css Auto-generated only. Run python3 build-tokens.py after updating tokens in Figma. Never edit this file by hand.

If these files drift out of sync with the documentation, Claude will build prototypes with outdated or missing components without any warning. Running git pull before each session is the simplest safeguard.
