Get started

Build realistic product screens that look exactly like the real app — by describing what you want in plain language, directly in Claude Code.

What is it

Build product screens with the real components and styles of this design system — by describing what you want in plain language.

The Engage Design System ships as a skill for Claude. Describe a screen in Dutch or English and Claude builds a working HTML prototype using the same components, colours and spacing as the real product. No Figma, no code.

For Effectory designers

The skill is pre-installed in the Effectory Claude account — just open Claude and start.

1
Open Claude with your Effectory account
Sign in to Claude using your Effectory login. You can use Claude in the browser, the desktop app, or Claude Code — the skill is available in all of them.
2
Describe what you want to build
Use words like prototype, design, scherm or pagina and Claude loads the skill automatically.
"bouw een prototype van het instellingenscherm" "maak een design voor een onboarding-flow met drie stappen" "prototype van een dashboard met filters en een tabel"
3
Iterate until it's right
Claude shows the result inline. Ask for tweaks — "make the button primary", "add a back link", "use dark mode" — and the file updates in place.

For everyone else

Outside Effectory? Install the skill from the public GitHub repo.

1
Download the skill from GitHub
The skill is open source. No GitHub account needed.
Download design system (.zip)
Unzip and save the folder somewhere easy to find. Or browse the source at github.com/jenteinsing123/effectory-design-documentation.
2
Install Claude Code
A free CLI tool by Anthropic. Open Terminal and run:
npm install -g @anthropic-ai/claude-code
No npm? Install Node.js first.
3
Run claude in the folder
Navigate to the unzipped folder and run claude. The skill in .claude/skills/ loads automatically. From here, follow the same flow as the Effectory team above.

Keep it fresh: re-download the ZIP before each session, or git pull if you cloned the repo.

Viewing your prototype

Open prototypes through the local server — double-clicking the HTML file breaks icons.

1
Start the local server
In Terminal, inside the project folder: python3 serve.py. Leave the window open while you work.
2
Open the link Claude gives you
Claude prints a URL like http://localhost:8000/prototypes/my-screen.html. Paste it into your browser.

How it works

Claude follows a few strict rules that keep prototypes consistent with the real product.

Only real components. Claude only uses components, colours, and icons that actually exist in the design system. It never invents something new or uses a rough approximation.
If something doesn't exist, Claude tells you. If you ask for a component that isn't in the system yet, Claude stops and tells you clearly — it won't silently build a substitute that looks different from the real product.
Prototypes look like the real product. Not a documentation page, not a wireframe — a realistic product screen. Dark mode and portal theming (Coordinator / Personal) work out of the box.
Files are saved automatically. Every prototype is saved in the prototypes/ folder inside the project. You can share the URL with teammates — as long as they're also running the local server.

For design system maintainers

Three files need to stay in sync with the documentation whenever components or tokens change.

File Update when
components.css A component is added or its CSS changes. Bundle the updated <style> block from the component's doc page into this file.
foundation.css Spacing, radius, shadow, or typography variables change. Mirrors the :root values from styles.css that prototypes need.
.claude/skills/effectory-design-system/design-system-reference.md A new component, variant, token, or icon is added. This is the reference Claude reads before building — if it's not listed here, Claude won't use it.
tokens.css Auto-generated only. Run python3 build-tokens.py after updating tokens in Figma. Never edit this file by hand.

If these files drift out of sync with the documentation, Claude will build prototypes with outdated or missing components without any warning. Running git pull before each session is the simplest safeguard.

Publishing the org-skill

The Effectory team uses the skill via the company Claude account, which runs a packaged copy of the skill. After any change to components.css, foundation.css or design-system-reference.md, rebuild and re-upload — otherwise the team keeps building with the old version.

1
Sync the bundle
In Terminal, from the project root, run ./sync-skill.sh. This copies the latest CSS, reference doc, icons and serve.py into skill-source/.
2
Build the zip
Run ./build-skill.sh. The output is dist/effectory-design-system.zip.
3
Upload to Claude.ai
Go to Claude.ai → Organization settings → Skills. Find effectory-design-system, choose update / re-upload, and pick the new zip. From that moment every Effectory teammate signed in to the company account uses the updated version automatically.
