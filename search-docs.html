Search

A specialised text input for filtering and finding content — always carries a leading search icon and uses type="search" for browser and assistive-technology support.

Anatomy

The search field is a single-line input with a fixed leading icon and an optional clear button on the trailing edge.

1Container
2Search icon
3Placeholder / value
1
Container — the bordered rectangle that carries the background, border, corner radius, and shadow. Matches the standard text field container.
2
Search icon — a fixed 16×16 magnifying-glass icon on the leading edge. Always present; not configurable.
3
Placeholder / value — muted placeholder text describing what to search for. Replaced by the user's query once they start typing.

Usage

Use a search field when users need to find items within a defined data set — a table, a list, or a page of results.

Do
Use a search field to filter a visible data set on the same page.
Don't
Don't use search fields as replacements for labelled form inputs.

Accessibility

→
Use type="search". This signals search semantics to assistive technology and enables browser-native behaviours like the clear button on iOS and history suggestions.
→
Always provide a label. Use a visible <label> where space allows. When the context makes the purpose obvious (e.g. a search bar at the top of a results page), use aria-label="Search" or role="search" on a wrapping landmark instead.
→
Mark the search icon as decorative. The magnifying-glass icon is cosmetic — hide it from the accessibility tree with aria-hidden="true" so screen readers don't announce it.
→
Announce results dynamically. When search filters a list, wrap the result count in a role="status" or aria-live="polite" region so screen readers announce the update without interrupting the user.

Accessibility

→
Always use type="search". This activates search-specific browser and AT behaviours — history, clear button on mobile, and the search landmark role in some implementations.
→
Provide a label. Use a visible <label> or an aria-label on the input. A generic "Search" label is fine only when the scope is obvious from surrounding context.
→
Hide the decorative icon. Add aria-hidden="true" to the magnifying-glass icon element so screen readers skip it.
→
Announce result counts live. Wrap the result count in role="status" or aria-live="polite" so updates are read out without moving focus.
→
For eff-search-box — label navigation buttons. The increment/decrement navigation buttons must have descriptive aria-label values such as "Next result" and "Previous result".
