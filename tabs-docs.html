Tabs

Switch between sections of the same view without leaving the page. Tabs keep related content in one place and surface one section at a time.

Anatomy

A row of tab labels sits above the content; the active tab is marked with a brand-coloured indicator and shows its panel below.

Usage

Use tabs to split one view's content into a few peer sections. Don't use them for steps in a flow (use a Stepper) or for navigating to different pages (use the navigation).

Accessibility

Tabs need the right roles and keyboard support so assistive tech announces and operates them as tabs.

→
Roles. The tab row is role="tablist", each tab role="tab", each panel role="tabpanel"; link them with aria-controls / aria-labelledby.
→
Selected state. The active tab carries aria-selected="true" — not just the visual underline.
→
Keyboard. Arrow keys move between tabs; only the active tab is in the tab order (tabindex roving).
→
Don't rely on colour alone. The underline plus selected state convey the active tab — colour isn't the only signal.
→
Disabled tabs. Mark them aria-disabled and skip them in keyboard navigation.

Usage

Use Angular Material's mat-tab-group with class="tabs". Each section is a mat-tab; its content is projected inside.

Basic

OverviewOpen answersReports
Content of the active tab.
<mat-tab-group class="tabs" mat-align-tabs="start" disableRipple>
  <mat-tab label="Overview">
    <div class="content"> … </div>
  </mat-tab>
  <mat-tab label="Open answers"> … </mat-tab>
  <mat-tab label="Reports"> … </mat-tab>
</mat-tab-group>

Icon label

For an icon beside the label, use a mat-tab-label template with an eff-mat-icon.

<mat-tab>
  <ng-template mat-tab-label>
    <eff-mat-icon [iconName]="'pie-chart'" class="tab-icon"/>
    <span>Reports</span>
  </ng-template>
  <div class="content"> … </div>
</mat-tab>

Accessibility

mat-tab-group wires up the tab roles and keyboard model; keep these in mind.

→
Roles + keyboard. Material sets tablist/tab/tabpanel, aria-selected and arrow-key navigation by default.
→
Label icons. If a tab is icon-only, give it an accessible label; prefer icon + text.
→
Disabled. Use disabled on mat-tab — it's skipped in keyboard navigation.
