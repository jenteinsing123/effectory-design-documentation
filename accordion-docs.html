Accordion

A stack of collapsible sections — each a header that expands to reveal its content. Use it to progressively disclose information and cut scrolling on long pages.

Anatomy

A bordered panel with a clickable header and a content area that shows when expanded.

This is the content of the panel. Keep it to what belongs under this section — a form step, details, or a longer explanation.
Hidden until expanded.
1
Panel — A bordered container for one section.
2
Header — The clickable row that toggles the section; holds the title.
3
Description — An optional second line under the title for extra context.
4
Chevron — Indicates expand/collapse; rotates when open.
5
Content — The body, revealed when the panel is open.

Usage

Use an accordion to chunk long content or forms into sections the user can open as needed. For switching between parallel views use Tabs; for a few always-visible options use a Segmented control.

Chunk long content

Do
Group related sections behind clear headers.
Don't
Hide essential, short content that should just be on the page.

Variants

Variations on the same panel — a leading icon, trailing metadata, a selection checkbox, or a borderless "hidden" header.

The default — a title and content.
Default — A header with a title and a content area. The most common form.
A leading icon helps scanning when sections have distinct types.
With icon — A leading icon before the title.
The description sits under the title, in secondary text.
With description — A second line under the title for extra context, in secondary text.
Counts and tags sit on the right of the header.
With trailing metadata — A count, tag or other meta on the right of the header. (This is what Figma calls the "Expansion panel".)
Accordion header
A checkbox in the header lets the user select the panel; the selected panel gets a brand-coloured border.
With selection — A checkbox in the header to select the panel. The selected panel is highlighted with a brand border.
No border and no side padding — just the title, chevron and content. For embedding inside a row or card that already has its own frame.
Borderless (hidden header) — Drops the panel border and side padding; shows only the title and chevron. Use it inside a container that already provides the frame.
⚠️
Figma gap — two names, one component Figma documents Accordion and Expansion panel as two patterns, but in the styleguide they're the same component (mat-expansion-panel). "Expansion panel" is just an accordion with a richer header (and a pop-out variant). Treat them as one.

Accessibility

An accordion is a set of expand/collapse buttons controlling regions.

→
Button header. The header is a <button> with aria-expanded reflecting open state and aria-controls pointing at the panel body.
→
Keyboard. Headers are in the tab order; Enter/Space toggles. Material's mat-expansion-panel wires this up.
→
Don't hide critical content. Anything essential or short should stay visible — collapsing adds a click and cognitive load.
→
Icon isn't the only cue. The chevron signals state, but aria-expanded carries it for assistive tech.

Usage

In production this is Angular Material's mat-expansion-panel (group several in a mat-accordion). The styleguide documents it as Accordions.

Single panel

This is the primary content of the panel.
<mat-expansion-panel>
  <mat-expansion-panel-header>
    <span class="text-l5">Example accordion</span>
  </mat-expansion-panel-header>
  <ng-template matExpansionPanelContent>
    <span>This is the primary content of the panel.</span>
  </ng-template>
</mat-expansion-panel>

Grouped (accordion)

Wrap panels in mat-accordion to group them; that's where single-open behaviour lives.

<mat-accordion>
  <mat-expansion-panel> … </mat-expansion-panel>
  <mat-expansion-panel> … </mat-expansion-panel>
</mat-accordion>
⚠️
Accordion = Expansion panel Figma splits this into "Accordion" and "Expansion panel", but the styleguide has one component: mat-expansion-panel. The richer "expansion panel" header (counts, tags, pop-out) is the same panel with more in its header.
