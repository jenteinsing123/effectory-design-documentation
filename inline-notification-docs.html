Inline notification

A persistent, non-blocking message that sits inline with page content to surface status, warnings, or contextual announcements.

Anatomy

An inline notification combines a status icon, a clear message, optional actions, and a way to dismiss.

Heads up! This alert needs your attention
1
Icon — A coloured icon that signals the message type (info, warn, success, error) at a glance. The subtle mood uses a sparkle.
2
Title — A short, scannable label that summarises the message in one or two words.
3
Message — The body text explaining what happened or what the user should do.
4
Close button — Lets the user dismiss the notification. Optional, controlled by showCloseButton.
+
Actions (optional) — An inline link/button or a dismiss button, placed before the close.

Usage

Use inline notifications when a message is tied to the surrounding content and the user should still see and interact with the page.

Choose the right mood

Well done!Your changes have been saved
Do
Match the mood to the situation — success for success, error for errors, warn for things needing attention, info for neutral context.
Oh snap!Your changes have been saved
Don't
Pair an error mood with a success message. The colour and tone send conflicting signals and erode trust.

Keep messages concise

Warning!Your subscription expires in 3 days
Do
Lead with the most important information. Title states what; message says what to do or expect next.
Warning!We've noticed that your subscription is scheduled to expire in three days from now and you may want to take action soon to avoid losing access
Don't
Pack the message with filler. Long-form prose belongs in a dialog or a dedicated page, not in an inline notification.

Variants

Five moods cover the most common signals; the one-liner layout handles short copy. Mood names match the Angular messageType input.

By action

A notification either stands on its own (close only) or carries one action button. An inline text button (inlineButtonText) or link (linkText) lives inside the message copy — add it when needed; it isn't a separate layout.

Secondary action button (buttonMessages) — A secondary button with a mood-coloured border (orange here for warn) — the "sentiment button". This is what the Angular styleguide renders today.

Primary action button — A button filled in the mood colour. This is the Figma intent for a high-emphasis action; the Angular styleguide does not expose it yet (see the Developer tab gap). Up to 3 action buttons can sit below the text (or right in a one-liner).

Content guidelines

Notifications are read at a glance. Keep titles short, messages specific, and actions verb-led.

  • Title (1–3 words) — A short label that mirrors the mood. Well done!, Warning, Heads up, New feature, Couldn't save.
  • Message (one sentence) — State what happened and, if relevant, what the user should do next. Avoid jargon and apologies.
  • Action label — Start with a verb that describes the outcome: Review, Try again, Learn more. Never Click here.
  • Punctuation — End full sentences with a period. Single-word titles can use an exclamation only when it adds meaning.

Accessibility

Inline notifications must be perceivable, operable, and announce themselves to assistive tech.

→
Pick the right role. Use role="status" for Subtle, Info, and Success notifications and role="alert" for Warn and Error. The styleguide renders role="alert" on the root .notification element.
→
Never colour alone. The mood colour is paired with an icon and a textual title; never rely on colour alone to communicate the type.
→
Label the close. Always include an aria-label like Dismiss notification. Focus must be visible — the :focus-visible ring is built in.
→
Keyboard reachable. The close button is reachable via Tab and activated with Enter or Space.
→
Contrast. All title and message text meets WCAG 2.2 AA (≥4.5:1) on the mood background.
→
Don't trap focus. Inline notifications are non-modal. Never move focus to them on mount unless the user just triggered an error they need to fix.

Usage

The notification is the Angular component <eff-inline-notification>. Set the mood with messageType and pass content via inputs.

Info — basic

Heads up!This alert needs your attention
<eff-inline-notification
  [messageTitle]="'Heads up!'"
  [message]="'This alert needs your attention'"
  [messageType]="'info'"
  [showCloseButton]="true"
  (closeNotification)="onClose()">
</eff-inline-notification>

Subtle — with inline text button

The inlineButtonText input renders a link-styled button inline within the message copy; inlineButtonCallback fires on click. We style it the same as an inline Link Inline. Project a sparkle icon for the subtle mood.

New feature!We released a new feature you might want to use. 
<eff-inline-notification
  [messageTitle]="'New feature!'"
  [message]="'We released a new feature you might want to use'"
  [messageType]="'subtle'"
  [inlineButtonText]="'Learn more'"
  (inlineButtonCallback)="onLearnMore()">
  <eff-mat-icon [iconName]="'featured'" subtleIcon class="icon subtle"></eff-mat-icon>
</eff-inline-notification>

Info — with link

Use linkText + linkHref for a navigating link, with linkIconPosition to place an optional icon.

Need a hand?Read the full walkthrough on our Support Page
<eff-inline-notification
  [messageTitle]="'Need a hand?'"
  [message]="'Read the docs for a full walkthrough'"
  [messageType]="'info'"
  [linkText]="'Support Page'"
  [linkHref]="'https://support.effectory.com/hc/en-us'"
  [linkIconPosition]="'after'">
</eff-inline-notification>

One-liner

Heads upYour trial ends soon
<eff-inline-notification
  [isOneLiner]="true"
  [messageTitle]="'Heads up'"
  [message]="'Your trial ends soon'"
  [messageType]="'warn'"

With sentiment buttons

Pass buttonMessages to render action buttons. Inside a notification these are secondary buttons whose border is automatically coloured to match the mood — the "sentiment button". Each entry gets its own (buttonCallback0), (buttonCallback1), … output.

Action requiredYour subscription expires in 3 days 
<eff-inline-notification
  [messageTitle]="'Action required'"
  [message]="'Your subscription expires in 3 days'"
  [messageType]="'warn'"
  [buttonMessages]="['Renew now']"
  (buttonCallback0)="onRenew()"
  [showCloseButton]="true">
</eff-inline-notification>

Accessibility

Pick the right role, label the close, and let users dismiss via keyboard.

→
Match the role to the mood. Warn and Error map to role="alert"; Subtle, Info, and Success map to role="status".
→
Label the close. The close button must have a descriptive aria-label (e.g. aria-label="Dismiss notification").
→
Don't steal focus. Keep focus order natural — don't move focus to the notification on mount unless it's a blocking error.
→
Live regions outside the dismiss button. If the message changes dynamically, place the live region outside the close button so screen readers don't re-announce on dismiss.
