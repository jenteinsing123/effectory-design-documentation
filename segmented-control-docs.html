Segmented Control

A compact group of two or three options where exactly one is selected — for switching views or modes within the same page.

Anatomy

A grey track holding equal segments; the selected segment is raised on a white card.

⚠️
Figma gap — different name This is called Segmented control in Figma, but the developer styleguide documents it as Button toggles on the Toggles page (mat-button-toggle). Same component, two names.
1
Track — The grey container that groups the segments.
2
Selected segment — White surface with a subtle shadow; marks the current choice.
3
Segment — An unselected option; flat on the track.
4
Icon — Optional 16px leading icon per segment.

Usage

Use a segmented control to switch between views of the same content — a chart vs a table, weekly vs monthly. For navigating to different content use Tabs; for settings that turn something on or off use a Toggle.

Few, parallel options

Do
Two or three equivalent views of the same data.
Don't
More than three options — use a Select instead.

Variants

Segments are either text only or paired with a leading icon. A control holds two or three segments — set that in code, not as a separate visual style.

Text only — The default. Use it when the labels are clear on their own.
With icon — A leading icon reinforces the option. Apply icons consistently: all segments or none.

States

A segment is either selected or not; unselected segments respond to hover.

Default
Hover (unselected)

Content guidelines

Segment labels are scanned side by side — keep them short and parallel.

One or two words, same form

Use short nouns in the same grammatical form, so the options read as equals.

Do
Short, parallel nouns.
Don't
Long or mixed phrasing.

Accessibility

A segmented control behaves as a radio group: one choice, always selected.

→
Radio semantics. Expose the group as role="radiogroup" with each segment as a radio (Material's button-toggle handles this), so the single-selection model is announced.
→
Keyboard. The group is one tab stop; arrow keys move between segments and select them.
→
Don't rely on the shadow alone. The selected state is carried by colour and elevation together; the label contrast must hold in both states.
→
Icons need labels. Every segment keeps a visible text label — icon-only segments aren't part of this component.

Usage

In production this is Angular Material's mat-button-toggle-group — the styleguide documents it under Toggles → Button toggles.

Single select

<mat-button-toggle-group>
  <mat-button-toggle [value]="'Red'">Red</mat-button-toggle>
  <mat-button-toggle [value]="'Green'">Green</mat-button-toggle>
  <mat-button-toggle [value]="'Blue'" checked>Blue</mat-button-toggle>
</mat-button-toggle-group>

With icons

<mat-button-toggle-group>
  <mat-button-toggle [value]="'Desktop'" checked>
    <eff-svg-desktop class="icon"></eff-svg-desktop>
    <span>Desktop</span>
  </mat-button-toggle>
  <mat-button-toggle [value]="'Mobile'">
    <eff-svg-mobile class="icon"></eff-svg-mobile>
    <span>Mobile</span>
  </mat-button-toggle>
</mat-button-toggle-group>

Multiple select

The styleguide also supports a multiple group where several segments can be on at once (e.g. Bold / Italic / Underline).

<mat-button-toggle-group multiple>
  <mat-button-toggle [value]="'Bold'" checked>Bold</mat-button-toggle>
  <mat-button-toggle [value]="'Italic'">Italic</mat-button-toggle>
  <mat-button-toggle [value]="'Underline'" checked>Underline</mat-button-toggle>
</mat-button-toggle-group>
⚠️
Figma gap — multiple select Figma's Segmented control only covers single select with 2 or 3 buttons. The styleguide's mat-button-toggle-group multiple (several segments on at once) has no Figma counterpart — design it before using it in new work.

Accessibility

Material's button-toggle wires up the group semantics; keep these in mind.

→
Group semantics. mat-button-toggle-group exposes the radio-group model (or checkbox model with multiple) automatically.
→
Always one selected. In single-select use, initialise with a checked segment — never render the group with nothing selected.
→
Visible labels. Each segment keeps its text label; icons are decoration, not replacement.
