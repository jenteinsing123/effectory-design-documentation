Checkbox

A binary toggle for selecting one or more options from a set. Four variants cover everything from a bare control in a table cell to a full card button in a filter bar.

Anatomy

The bare checkbox is three layers — a container, a selection indicator (check or dash), and an optional border ring. The label, description, and button variants wrap this control with additional content.

1Container
2Indicator
1
Container — 16×16 px square (20×20 in large size), 4 px radius. Unchecked: white background + 1.5 px border/action border. Checked or indeterminate: filled bg/brand/base with no visible border.
2
Indicator — a white check SVG when checked, a white dash SVG when indeterminate. In the "select all" variant the container stays white and the indicator is teal instead of white. Hidden when unchecked.

Usage

Checkboxes let users make independent binary choices. Use them when selections are non-exclusive — multiple items can be chosen at once. For a single on/off toggle, use the Toggle instead.

Use when
  • Users can select zero, one, or many items from a list
  • Each option is independent — selecting one doesn't affect others
  • Options need a label and optionally a short description
  • You're building a filter row and need compact togglable chips
Avoid when
  • Only one option can be active at a time — use Radio Group
  • The action takes effect immediately without a "Save" step — use Toggle
  • You have more than ~8 options — consider a multi-select Select

Best practices

Do
Give every checkbox a visible text label that describes exactly what selecting it does.
Don't
Never render checkboxes without labels — screen readers will announce the control with no context, and sighted users won't know what they're selecting.
Do
Show the indeterminate state on a "select all" checkbox when only some children are selected — it tells users the list is partially checked.
Don't
Don't leave the parent checkbox unchecked when children are partially selected — the state is ambiguous and users can't tell whether any items are chosen.

Variants

Four variants cover the full range of checkbox use cases — from bare controls in data tables to full-surface button chips in filter bars.

Checkbox — the bare control. Default size is 16×16 px; Large is 20×20 px. Use in table cells, custom layouts, or anywhere you supply your own label markup.
Labeled — wraps the control with a right-aligned text label. The label and checkbox share a single click target — always use a semantic <label> element.
Label Subtitle with a short explanation.
With description — adds a secondary description line below the label. Responds to desktop and mobile widths. An inline link can optionally be appended to the description text.
Chip — a pill-shaped card where the entire surface is the tap target. Used in compact horizontal filter bars or tag pickers. The embedded checkbox indicator shows selection state; no separate label element needed outside the button.

Accessibility

→
Always use a native <input type="checkbox">. You get keyboard focus, Space to toggle, and correct announcements ("checkbox, checked") for free.
→
Link every checkbox to its label. Use a wrapping <label> or pair id on the input with for on the label — both expand the tap target and make the association audible to screen readers.
→
Programmatic indeterminate only. The indeterminate state has no HTML attribute — set it in JavaScript with input.indeterminate = true. Screen readers announce it as "mixed".
→
Group related checkboxes in a <fieldset> with a <legend>. The legend becomes the group label that screen readers read before each option.
→
The Chip still needs a semantic input. The <label> pill wraps a real <input type="checkbox"> — don't replace it with a <div role="checkbox">. The native element handles keyboard and AT for free.

Usage

Import EffCheckboxModule and use the <eff-checkbox> component. The component renders a native <input type="checkbox"> internally and handles all accessibility wiring.

Bare checkbox

<!-- Unchecked -->
<eff-checkbox
  [checked]="false"
  (checkedChange)="onChecked($event)"
></eff-checkbox>

<!-- Checked -->
<eff-checkbox
  [checked]="true"
  (checkedChange)="onChecked($event)"
></eff-checkbox>

Labeled

<eff-checkbox
  [checked]="true"
  [labelText]="'Send me weekly summary emails'"
  (checkedChange)="onChecked($event)"
></eff-checkbox>

<!-- Disabled -->
<eff-checkbox
  [checked]="false"
  [disabled]="true"
  [labelText]="'Disabled option'"
></eff-checkbox>

With description

The With description variant wraps <eff-checkbox> with your own label and description markup alongside it. The checkbox component itself only covers the control and its inline label text.

<eff-checkbox
  [checked]="isManagerVisible"
  [labelText]="'Manager visibility'"
  (checkedChange)="isManagerVisible = $event"
></eff-checkbox>
<p class="checkbox-description">Managers can view their team's anonymised results.</p>

Select all with intermediate

Use [intermediate] (note: not [indeterminate]) to show a dash when a subset of items is selected. The component maps this to the native indeterminate state internally.

<eff-checkbox
  [checked]="allChecked"
  [intermediate]="someChecked && !allChecked"
  [labelText]="'All surveys'"
  (checkedChange)="toggleAll($event)"
></eff-checkbox>

<!-- Child items -->
<eff-checkbox
  *ngFor="let survey of surveys"
  [checked]="survey.selected"
  [labelText]="survey.name"
  (checkedChange)="survey.selected = $event; updateParent()"
></eff-checkbox>

API reference

NAMETYPEDEFAULTDESCRIPTION
[checked]booleanfalseWhether the checkbox is checked.
[intermediate]booleanfalseShows the indeterminate (dash) state. Use when a subset of child options is selected. Note the spelling — the input is intermediate, not indeterminate.
[disabled]booleanfalseDisables the control and applies muted styling. Removes the element from the tab order.
[labelText]stringundefinedInline label rendered next to the control. Omit for a bare checkbox.
(checkedChange)EventEmitter<boolean>—Emits the new checked value whenever the user toggles the control.

Accessibility

→
The component handles native semantics. <eff-checkbox> renders a real <input type="checkbox"> internally — keyboard interaction, AT announcements, and form participation work out of the box.
→
Use [labelText] for the accessible label. This wires the visible text to the internal input so screen readers announce it on focus. If you need richer label content, ensure the input still has an accessible name via aria-label on a wrapper.
→
[intermediate] announces as "mixed". Assistive technologies read the indeterminate state as "mixed" — use it only for genuine partial-selection scenarios like select-all controls.
→
Group related checkboxes with <fieldset> + <legend>. The legend names the group — screen readers read it before each option so users always know what they're selecting.
→
Disabled removes the item from tab order. Setting [disabled]="true" applies the disabled attribute to the underlying input — the control is skipped during keyboard navigation and excluded from form submissions.
