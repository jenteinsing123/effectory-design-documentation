Radio Group

Radio buttons let users select exactly one option from a set. Two display variants are available: a standard labeled list and a card-style tile layout.

Anatomy

The radio button pairs a circular control with a required label. The tile variant adds a title, description, and a full-surface click target.

1Container
3Label
Label
2Indicator
1
Container — circular control box. Unselected: white background with an action border. Selected: filled with the brand colour. Hover darkens the border; disabled reduces opacity.
2
Indicator — a white dot centred in the container, visible only when the option is selected.
3
Label — required text to the right of the container, vertically centred with it. Appears muted when disabled.
4
Description (tile variant only) — secondary text below the label, providing additional context. The entire tile surface is the click target.

Usage

Radio buttons enforce a single selection from a mutually exclusive set. Always present all options at once — never hide options behind a load action.

Use when
  • Users must choose exactly one option from a small set (2–7 items)
  • All options need to be visible simultaneously for comparison
  • The default selection is meaningful and should be pre-selected
  • Options are mutually exclusive — selecting one deselects the rest
Avoid when
  • Multiple selections are valid — use Checkbox instead
  • There are more than ~7 options — use Select
  • It's a single binary on/off choice — use Toggle
  • Options are too similar and need more context — use Radio tiles

Choosing a variant

VARIANTWHEN TO USE
Radio groupStandard form fields, settings, and filters. Labels are short (1–4 words). Compact vertical or horizontal layout.
Radio tilesWhen each option needs a short description to be understood — plan selection, onboarding flows, survey question types. The larger click target also helps on touch screens.

Best practices

Do
Always pre-select a sensible default. A group with nothing selected forces the user to make a choice before they can continue.
Don't
Don't leave the group with nothing selected on load. If no default is appropriate, add an explicit "None" option.
Anonymous
Responses are collected without identifying information.
Confidential
Visible to HR only, not to direct managers.
Do
Use Radio tiles when options need a description to distinguish them clearly.
Don't
Don't stuff a label and description into a single label string — use Radio tiles instead so the hierarchy and click target are correct.

Variants

Two layout variants cover the full range of single-select use cases.

Radio group

A vertical list of labeled options. The standard choice for form fields, settings, and filters with 2–7 short options.

Use for: frequency settings, single-select filters, short option lists

Value 1
Short description of what this option means.
Value 2
Short description of what this option means.

Radio tiles

Card-style options with a title and supporting description. The full tile surface is the click target.

Use when options need more context to compare, or when the choice carries more weight.

Radio group

Use Angular Material's mat-radio-group with mat-radio-button children. Bind the group value with [(ngModel)] or reactive forms.

Basic

<mat-radio-group
  [(ngModel)]="selectedValue"
  name="my-group"
>
  <mat-radio-button value="option1">Option 1</mat-radio-button>
  <mat-radio-button value="option2">Option 2</mat-radio-button>
  <mat-radio-button value="option3" [disabled]="true">Option 3</mat-radio-button>
</mat-radio-group>

With reactive forms

<!-- Template -->
<mat-radio-group [formControl]="deliveryControl">
  <mat-radio-button value="immediate">Send immediately</mat-radio-button>
  <mat-radio-button value="scheduled">Schedule for later</mat-radio-button>
  <mat-radio-button value="draft">Save as draft</mat-radio-button>
</mat-radio-group>

// Component
deliveryControl = new FormControl('scheduled');

Radio tiles

Wrap each mat-radio-button in a .radio-tile container and add the no-label class to the button. Connect the tile label via aria-labelledby for accessibility.

Anonymous
Responses are collected without any identifying information attached to the submission.
Confidential
Visible to HR only — not surfaced to direct managers or team leads.
Public
Results published on the company page (not available on your plan).
<mat-radio-group [(ngModel)]="visibility" name="visibility">

  <div class="radio-tile">
    <mat-radio-button
      class="no-label"
      id="anon"
      value="anonymous"
      aria-labelledby="anon-label"
    ></mat-radio-button>
    <label id="anon-label">
      <strong>Anonymous</strong>
      <p>Responses collected without identifying information.</p>
    </label>
  </div>

  <div class="radio-tile">
    <mat-radio-button
      class="no-label"
      id="conf"
      value="confidential"
      aria-labelledby="conf-label"
    ></mat-radio-button>
    <label id="conf-label">
      <strong>Confidential</strong>
      <p>Visible to HR only.</p>
    </label>
  </div>

</mat-radio-group>

Disabled states

Disable the entire group with [disabled] on mat-radio-group, or disable individual options with [disabled] on a specific mat-radio-button.

<!-- Disable the whole group -->
<mat-radio-group [(ngModel)]="value" [disabled]="isDisabled">
  <mat-radio-button value="a">Option A</mat-radio-button>
  <mat-radio-button value="b">Option B</mat-radio-button>
</mat-radio-group>

<!-- Disable a single option -->
<mat-radio-group [(ngModel)]="value">
  <mat-radio-button value="a">Option A</mat-radio-button>
  <mat-radio-button value="b" [disabled]="true">Option B</mat-radio-button>
</mat-radio-group>

API reference

The Radio Group uses Angular Material. Both mat-radio-group and mat-radio-button are from @angular/material/radio.

mat-radio-group

INPUT / OUTPUTTYPEDEFAULTDESCRIPTION
[(ngModel)]any—Two-way binding for the selected value. Alternatively use [formControl] with reactive forms.
[value]anynullThe currently selected value (one-way). Use [(ngModel)] or reactive forms for two-way binding.
[name]stringauto-generatedName attribute forwarded to all child radio inputs. Must be unique per page.
[disabled]booleanfalseDisables all radio buttons in the group.
[required]booleanfalseMarks the group as required for form validation.
(change)MatRadioChange—Emits when the selected value changes. Payload: { source: MatRadioButton, value: any }.

mat-radio-button

INPUT / OUTPUTTYPEDEFAULTDESCRIPTION
[value]any—The value this button represents. Compared against the group's value to determine selection.
[disabled]booleanfalseDisables this specific option without affecting the rest of the group.
