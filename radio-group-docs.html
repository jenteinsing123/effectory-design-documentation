Radio Group

Radio buttons let users select exactly one option from a set. Six display variants are available: a bare control, a labeled option, a described option, a card-style tile, and compact pill buttons in ghost and bordered styles.

Anatomy

The radio button pairs a circular control with a required label. The tile variant adds a title, description, and a full-surface click target.

Container — circular control box. Unselected: white background with an action border. Selected: filled with the brand colour. Hover darkens the border; disabled reduces opacity.

Indicator — a white dot centred in the container, visible only when the option is selected.

Label — required text to the right of the container, vertically centred with it. Appears muted when disabled.
4
Description (tile and with-description variants) — secondary text below the label, providing additional context. In the tile variant the entire card surface is the click target.

Usage

Radio buttons enforce a single selection from a mutually exclusive set. Always present all options at once — never hide options behind a load action.

Use when
  • Users must choose exactly one option from a small set (2–7 items)
  • All options need to be visible simultaneously for comparison
  • The default selection is meaningful and should be pre-selected
  • Options are mutually exclusive — selecting one deselects the rest
Avoid when
  • Multiple selections are valid — use Checkbox instead
  • There are more than ~7 options — use Select
  • It's a single binary on/off choice — use Toggle
  • Options need extra context — use the With description or Radio tiles variant

Content guidelines

Radio option labels name the choice — not the act of selecting it.

Use noun phrases or short clauses

Options should read as things to choose from, not instructions. Avoid imperative verbs ("Select monthly", "Choose admin").

Do
Noun phrases scan quickly and feel like genuine options, not commands.
Don't
Imperative labels ("Choose", "Select") are redundant — the radio button already signals that.

Keep all options parallel

Every option in a group should use the same grammatical form and level of detail. Mixing styles forces users to process each option differently instead of scanning the list.

Do
Single-word role names — consistent, scannable, parallel.
Don't
Wildly different lengths and styles across the group break visual rhythm.

Accessibility

→
Always use a native <input type="radio">. You get keyboard navigation, arrow-key selection between options, and correct AT announcements ("radio button, checked") for free. Never replace a real radio with a <div role="radio">.
→
Group related radios in a <fieldset> with a <legend>. The legend becomes the group label that screen readers announce before each option, giving users full context for the choice they're making.
→
Always pre-select a sensible default. A group with nothing checked on load forces the user to make an extra decision. If no option is truly a default, add an explicit "None" option and pre-select it.
→
Use aria-labelledby for tiles and described options. When the visible label is a sibling element rather than a wrapping <label> (as in the Radio tile and With description patterns), connect the input to its label via aria-labelledby.
→
Arrow keys move between options; Tab exits the group. Once a radio group has focus, users navigate with arrow keys — not Tab. Tab moves focus out of the group entirely. Do not override or suppress this native behaviour.

Usage

Use Angular Material's mat-radio-group with mat-radio-button children. Bind the group value with [(ngModel)] or reactive forms.

Bare radio

<!-- Unchecked -->
<mat-radio-button
  class="no-label"
  value="a"
></mat-radio-button>

<!-- Checked -->
<mat-radio-button
  class="no-label"
  value="b"
  [checked]="true"
></mat-radio-button>

Sizes

Two sizes: default (16×16 px) and large (20×20 px). Use large when the radio is the primary interactive element or the surrounding UI uses larger touch targets.

16×16 px
20×20 px 
<!-- Default (16×16 px) -->
<mat-radio-button value="a">Default</mat-radio-button>

<!-- Large (20×20 px) — proposed, not yet in Figma -->
<mat-radio-button class="rb-lg" value="b">Large</mat-radio-button>

Labeled

<mat-radio-group
  [(ngModel)]="selectedValue"
  name="my-group"
>
  <mat-radio-button value="option1">Option 1</mat-radio-button>
  <mat-radio-button value="option2">Option 2</mat-radio-button>
  <mat-radio-button value="option3" [disabled]="true">Option 3</mat-radio-button>
</mat-radio-group>

With description

Pair each mat-radio-button with a description element. Use aria-labelledby to associate the button with its label text for screen reader accessibility.

<mat-radio-group [(ngModel)]="delivery">
  <div class="radio-info">
    <mat-radio-button
      class="no-label"
      id="send-now"
      value="immediate"
      aria-labelledby="send-now-label"
    ></mat-radio-button>
    <label id="send-now-label">
      <strong>Send immediately</strong>
      <p>The survey goes live as soon as you confirm.</p>
    </label>
  </div>
</mat-radio-group>

Disabled states

Disable the entire group with [disabled] on mat-radio-group, or disable individual options with [disabled] on a specific mat-radio-button.

<!-- Disable the whole group -->
<mat-radio-group [(ngModel)]="value" [disabled]="isDisabled">
  <mat-radio-button value="a">Option A</mat-radio-button>
  <mat-radio-button value="b">Option B</mat-radio-button>
</mat-radio-group>

<!-- Disable a single option -->
<mat-radio-group [(ngModel)]="value">
  <mat-radio-button value="a">Option A</mat-radio-button>
  <mat-radio-button value="b" [disabled]="true">Option B</mat-radio-button>
</mat-radio-group>

Accessibility

→
The component handles native semantics. mat-radio-group and mat-radio-button render a real <input type="radio"> internally — keyboard interaction, AT announcements, and form participation work out of the box.
→
Always label the group. Wrap the radio group in a <fieldset> with a <legend>, or set aria-labelledby on mat-radio-group pointing to a visible heading. Screen readers announce the group label before each option.
→
Pre-select a default value. Set an initial value on [(ngModel)] or the FormControl so no group is left empty on load. If no default is meaningful, add an explicit "None" option and pre-select it.
→
Use aria-labelledby for tiles and descriptions. When the visible label is a sibling element (as in Radio tiles and With description), add aria-labelledby on the mat-radio-button pointing to the label element's id.
→
Disabled removes the item from tab order. Setting [disabled]="true" applies the disabled attribute to the underlying input — the control is skipped during keyboard navigation and excluded from form submissions.
