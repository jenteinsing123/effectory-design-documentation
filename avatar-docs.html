Avatar

A small circular surface that represents a person. Shows initials when we know the name; falls back to an anonymous icon when we don't — or when the person must stay anonymous.

Anatomy

Two parts: a circular container and the content inside — either initials or an icon.

Usage

Avatars represent one person at a single point in the UI. They are not status indicators or category badges — when in doubt, use a Tag or Badge instead.

Variants

The Type property switches between the two forms. The Size property scales it; Colour picks the accent tint.

Type

JI
Initials — the default. Use whenever the person's display name is available. Two characters max, derived from the first letter of their first and last name.
Icon — the anonymous form. Use when the name is genuinely unknown, or when anonymity is required (anonymous survey responses, redacted authors). Always rendered grey.

JI

Colour

Five named backgrounds available via [background]. For per-person identity colours, use [enableDynamicColor]="true" instead — the component then derives the tint from the avatar's text. Grey is reserved for the anonymous and counter forms and is never user-pickable.

JI
info — blue. Neutral identity colour, the default when no theme is needed.
JR
brand — turquoise. Use when the avatar represents an Effectory entity (e.g. the system, a default coach).
MY
warning — orange. Reserved for avatars tied to a warning state, never for plain people.
SS
negative — red. Reserved for avatars in a destructive or error context.
JT
accent — yellow. Highlights, badges of honour, featured authors.
AA BI CJ DF EM FC
enableDynamicColor — the component picks one of six rotating tints (info, brand, green, warning, negative, accent) by hashing the avatar's text. Same person → same colour, everywhere.
Grey — applied automatically when the avatar's text is empty (anonymous) or contains an overflow counter (+3). Not exposed as a user-pickable option.

States

Avatars are non-interactive by default. They become interactive only when wrapped in a button or link — in which case the surrounding control handles the state, not the avatar itself.

Accessibility

→
Always pass the full name as tooltip. The component exposes it as the accessible name, so screen readers announce who the avatar represents.
→
Anonymous keeps tooltip empty. When text is empty the icon variant renders; an empty tooltip is correct — there's no name to announce.
→
Colour is not the identity. Screen readers don't announce avatar colour, and 8% of users can't distinguish all six tints reliably. Never rely on colour alone to convey who someone is.
→
Contrast. Each subtle background pairs with its matching base content colour — all combinations meet WCAG AA for text at 14 px and up.

Usage

The Angular component <eff-avatar> takes an IAvatar object plus a size and colour. Set enableDynamicColor to derive the colour from the avatar text instead of providing one yourself.

Fixed background

JI JR MY SS JT 
<eff-avatar [avatar]="{ text: 'JI', tooltip: 'Jamie Insing' }" [background]="'info'" [size]="'m'" />

<eff-avatar [avatar]="{ text: 'SS', tooltip: 'Sam Smith' }" [background]="'negative'" [size]="'l'" />

Dynamic colour (per-person identity)

AA BI CJ DF EM FC 
<eff-avatar [avatar]="{ text: 'AA', tooltip: 'Alice Anderson' }" [enableDynamicColor]="true" />
<eff-avatar [avatar]="{ text: 'BI', tooltip: 'Bob Ingram' }" [enableDynamicColor]="true" />
<eff-avatar [avatar]="{ text: 'CJ', tooltip: 'Charlie Jones' }" [enableDynamicColor]="true" />

Anonymous (no text)

// When IAvatar.text is empty the component renders the anonymous icon variant.
<eff-avatar [avatar]="{ text: '', tooltip: '' }" [size]="'m'" />

Counter (overflow inside Avatar Group)

+3 +12 +5 
// Pass a numeric overflow string as text. The grey surface is applied automatically.
<eff-avatar [avatar]="{ text: '+3', tooltip: '3 more' }" [size]="'m'" />

Accessibility

<eff-avatar> renders the tooltip string as the accessible name. Pass the full display name (e.g. "Jamie Tan") so screen readers announce who the avatar represents.

// Tooltip carries the accessible name — always use the full display name.
<eff-avatar [avatar]="{ text: 'JT', tooltip: 'Jamie Tan' }" />

// Anonymous avatar — keep tooltip empty; screen readers announce nothing,
// which is the correct behaviour for an unidentified person.
<eff-avatar [avatar]="{ text: '', tooltip: '' }" />
