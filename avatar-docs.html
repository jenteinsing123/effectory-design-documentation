Avatar

A small circular surface that represents a person. Shows initials when we know the name; falls back to an anonymous icon when we don't — or when the person must stay anonymous.

Anatomy

Two parts: a circular container and the content inside — either initials or an icon.

Usage

Avatars represent one person at a single point in the UI. They are not status indicators or category badges — when in doubt, use a Tag or Badge instead.

Variants

The Type property switches between the two forms. The Size property scales it; Colour picks the accent tint.

Type

JI
Initials — the default. Use whenever the person's display name is available. Two characters max, derived from the first letter of their first and last name.
Icon — the anonymous form. Use when the name is genuinely unknown, or when anonymity is required (anonymous survey responses, redacted authors). Always rendered grey.

Size

JI
40 px — profile menus, settings pages, user-detail headers.
JI
36 px — top-bar account chip, action-bar leading avatars.
JI

JI
24 px — dense tables, comment metadata, breadcrumb trails.
JI
20 px — smallest stacks (Avatar Group with many overflow members), notification list rows.

Colour

Seven colour options. Six accent colours are assigned per person for recognisability; grey is reserved for the anonymous icon variant and overflow counters.

Grey — reserved for the anonymous icon variant and overflow counters (+3). Never assign grey to a named person.

States

Avatars are non-interactive by default. They become interactive only when wrapped in a button or link — in which case the surrounding control handles the state, not the avatar itself.

Accessibility

An avatar is decorative when the person's name appears next to it; it carries meaning on its own only in dense rosters.

Decorative by default. When the avatar sits next to a visible name label, mark it aria-hidden="true". The name carries the meaning.
Standalone needs a label. If the avatar appears without a visible name (e.g. compact tables), add aria-label="Jamie Tan" with the full name.
Colour is not the identity. Screen readers don't announce avatar colour, and 8% of users can't distinguish all six tints reliably. Never rely on colour alone to convey who someone is.
Contrast. Each subtle background pairs with its matching base content colour — all combinations meet WCAG AA for text at 14 px and up.

Accessibility

Mark the avatar as decorative when a name label is visible nearby; give it an aria-label when it stands alone.

<!-- Next to a visible name — decorative -->
<span class="av av-32 av-blue" aria-hidden="true">JT</span>
Jamie Tan

<!-- Standalone — needs a label -->
<span class="av av-32 av-blue" role="img" aria-label="Jamie Tan">JT</span>
