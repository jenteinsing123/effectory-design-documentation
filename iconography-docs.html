Foundations Stable

Iconography

Engage uses a single line-icon set — consistent stroke width, square optical alignment, and three fixed sizes. Icons carry meaning at a glance: pair them with a label wherever the meaning isn't obvious, and let them stand alone only for universally understood actions.

📐 Sizes

Three fixed sizes. Pick the one that matches the surrounding text size — icons and text should feel balanced, not dominant.

16 px Inline with body text, dense lists, compact controls.
24 px Default. Buttons, nav items, menu rows, toolbars.
32 px Empty states, feature highlights, large header actions.
Stroke stays at 2 px across all sizes. Don't rescale the SVG stroke-width — it breaks visual consistency against text and against other icons on the same surface.

🧩 Usage

Icons earn their place when they clarify — and get in the way when they don't. A few patterns that hold up everywhere.

Icon + label in buttons
Use a 16 px icon with spacing-extra-tight to the label. The icon reinforces the action — the label carries the meaning.
Icon-only controls
Reserve for universally understood actions (edit, share, delete). Always include an aria-label — there's no text to fall back on.

✨ Guidance

Meaning first, decoration never

Every icon should clarify something the label alone can't. If you added it for visual flavour, remove it — decorative icons add noise to the scan.

One icon per meaning

The same action always gets the same icon. If edit means "edit" in one place, don't use pen-tool elsewhere. Consistency builds the visual vocabulary users rely on.

Match the size to the context

16 px next to body text, 24 px in buttons and menu rows, 32 px in empty states. Don't free-scale — if nothing fits, the container is likely the wrong size.

Always label icon-only buttons

An icon without a visible label still needs an aria-label and a tooltip. Assistive tech and first-time users both lose if the meaning lives only in the glyph.
