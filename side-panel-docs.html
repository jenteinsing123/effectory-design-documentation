Side Panel

A panel that slides in from the right over a scrim to show details, a form, or a focused task without leaving the current page.

Anatomy

A side panel docks against the right edge at full height, over a scrim. A header holds the title and actions; the body scrolls; an optional footer pins the actions.

Usage

Use a side panel to view or edit a record beside its context — the list stays visible behind the scrim. For a full, standalone task use a page; for a short confirmation use a Dialog.

Header size

A Large header (26px title) puts the toolbar on its own row above the title; a Small header (18px title) is more compact and places the close/actions on the title row. Try both in the playground.

Footer

A footer is optional. Add one when the panel edits data and needs explicit Save/Cancel actions; omit it for read-only detail panels (use the close button).

Content guidelines

Give the panel a clear title and keep its content and actions focused.

Write a specific title

The title should say what the panel is about. Avoid generic labels like "Details" or "Information"; a short subtitle can add context.

Accessibility

A side panel is a modal surface — manage focus and dismissal like a dialog.

→
Modal semantics. Expose role="dialog" + aria-modal="true" and name it via aria-labelledby on the title.
→
Focus. Move focus into the panel on open, trap Tab within it, and return focus to the trigger on close.
→
Escape closes. Escape dismisses the panel (= close), unless an edit is mid-flight.
→
Label icon actions. The toolbar and nav arrows are icon-only — give each an aria-label (e.g. "Previous", "Bookmark", "Close").
→
Tabs. If the header uses tabs, wire them with role="tab" / role="tabpanel" and arrow-key navigation.

Usage

Open a panel by passing a component (and optional data) to the SidepanelService. The panel's own template renders the title, content and actions.

Open & close

// component that opens the panel
import { SidepanelService } from "@effectory/branding-angular";

constructor(private sidepanelService: SidepanelService) {
  this.sidepanelService.afterClosed
    .pipe(takeUntilDestroyed(this.destroyRef))
    .subscribe(data => this.result = data);
}

open() {
  this.sidepanelService.open(ResponseSidepanelComponent, { name: "Said" });
}

The panel component

<h1 class="title text-l3">{{ data.name }}</h1>
<span class="subtitle">Response details</span>
<button class="icon-button close-button" (click)="sidepanelService.close()">
  <eff-mat-icon class="icon" [iconName]="'cross'"/>
</button>

<p>The content goes here</p>

<div class="actions-container">
  <button class="secondary" (click)="sidepanelService.close()">Cancel</button>
  <button class="primary" (click)="save()">Confirm</button>
</div>

Return data to the opener by passing it to close() — e.g. sidepanelService.close({ saved: true }) — and read it through afterClosed. A narrower panel opens via the service's small variant.

→

→
Focus return. Focus returns to the trigger when the panel closes.
→
Icon labels. Give every icon-only toolbar/nav button an aria-label.
