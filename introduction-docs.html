Getting started

Three steps to get up to speed, wherever you're starting from.

1
Read the Design Principles
The Design Principles explain the thinking behind the system — the values and trade-offs that inform every decision, and the tiebreaker when two valid options exist.
2
Learn the token system
Spend 10 minutes with Colors and Spacing. Understanding how semantic tokens map to raw values makes every component page easier to read and makes building new screens much faster.
3
Open any component page
Each page has a Design tab (anatomy, usage guidelines, variants, states, tokens) and a Developer tab (live playground, Angular API, copy-ready code). If something is designed but not yet built, a ⚠ Figma gap callout says so explicitly.
