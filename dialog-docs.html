Dialog

A focused modal surface shown over a scrim to confirm a decision, complete a short task, or surface a message that needs the user's full attention.

Anatomy

A dialog is a card centred over a dimmed scrim. It holds a title, optional supporting text, the content, and a footer with the actions.

Header

An explanation of some sorts — what this dialog is for and what happens next.

Usage

Use a dialog for a short, focused interruption the user must resolve before continuing. Don't use it for long flows, non-urgent info, or anything that fits inline.

One primary action

One clear primary action plus a way out (Cancel or close).

Don't
Stack several competing primaries — the user can't tell what's expected.

Don't
Block the page for non-urgent tips — use a Spotlight, Toast, or inline message.

Width scales with how much the content needs; height always fits the content. Pick the smallest that comfortably holds your content.

Content guidelines

A dialog should be readable in seconds: a specific title, lean supporting text, and action labels that say what they do.

Title states the decision

Name the task or ask the question directly. Avoid vague titles like "Are you sure?".

Accessibility

A dialog takes over the page, so focus and dismissal must be handled rigorously.

→
Modal semantics. Mark the surface with role="dialog" and aria-modal="true" so assistive tech treats the rest of the page as inert.
→
Name it. Point aria-labelledby at the title (and aria-describedby at the supporting text) so the dialog is announced when it opens.
→
Trap and restore focus. Move focus into the dialog on open, keep Tab cycling within it, and return focus to the trigger on close.
→
Escape closes. Escape dismisses the dialog (equivalent to Cancel / close), unless a destructive action is mid-flight.
→
Scrim click. Clicking the scrim may close non-destructive dialogs; for ones with unsaved input, confirm first rather than discarding silently.
→
Don't autofocus destructive actions. Put initial focus on a safe control (Cancel or the first field), not on "Delete".

Usage

Define the content as an <ng-template> built from Card classes, then open it with MatDialog. Size is a panelClass; the scrim is backdropClass="dialog-backdrop".

Template + open

<!-- template -->
<ng-template #cardDialog let-dialogRef="dialogRef">
  <div class="card">
    <div class="card-title-container">
      <h3 class="text-l5 card-title">Discard changes?</h3>
      <button class="icon-button close-button" (click)="dialogRef.close()" [attr.aria-label]="'shell.dialog-close-button-label' | translate">
        <eff-mat-icon class="icon small-icon" [iconName]="'cross'"/>
      </button>
    </div>
    <div class="card-content-container">You have unsaved edits to this report.</div>
    <div class="action-container">
      <button class="action-button secondary" (click)="dialogRef.close()">Keep editing</button>
      <button class="action-button primary" (click)="dialogRef.close()">Discard</button>
    </div>
  </div>
</ng-template>

// component.ts
@ViewChild("cardDialog") cardDialog!: TemplateRef<any>;
constructor(public matDialog: MatDialog) {}

openDialog() {
  const config = new MatDialogConfig();
  config.panelClass = "dialog-s";          // dialog-xs | dialog-s | dialog-m | dialog-l
  config.backdropClass = "dialog-backdrop";
  this.matDialog.open(this.cardDialog, config);
}

Title size & icon

Use card-title (18px, with text-l5) or card-title-large (26px, with text-l3). For a leading icon add icon-before to the eff-mat-icon inside the title container.

<div class="card-title-container">
  <eff-mat-icon class="icon icon-before" [iconName]="'info'"/>
  <h3 class="text-l3 card-title-large">New: team filters</h3>
</div>

Visual header & footer shadow

Add a .card-illustration-container for a hero image, and .with-shadow on .action-container when the body scrolls under the footer. Centred actions: add "center-action-btns" to panelClass. Fullscreen variants use "dialog-fullscreen" with with-header / with-footer / with-progress-bar (Material mat-dialog-content / mat-dialog-actions).

→
role + aria-modal. Set role="dialog" and aria-modal="true"; mark background content inert.
→
Label. aria-labelledby → the title id; aria-describedby → the supporting text.

→
Escape. Closes the dialog (= Cancel), except mid destructive action.
