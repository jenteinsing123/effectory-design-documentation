Selection Button

A compact control that displays a label and its currently selected value. Tapping it opens a dropdown or popover to change the selection. Use it in toolbars and filter bars to give users quick access to a single filter or sort option.

Anatomy

Up to five parts. Only the name and value are always required — the rest are optional depending on context.

1Container
2Icon
3Name
4Value
1
Container — carries the border, background, and shadow in the bordered appearance; invisible in the inline appearance.
2
Icon (optional) — a 16×16px leading icon, typically a filter or funnel symbol, to reinforce the button's role at a glance.
3
Name — the label for the filter or sort, rendered in SemiBold. Should end with a colon: "Filter:", "Sort:", "Status:".
4
Value — the currently selected option, rendered in Medium weight. Updates whenever the user changes their selection.
5
Chevron (optional) — a trailing chevron-down that signals a dropdown or popover will appear on click.

Usage

Selection Buttons surface a single active filter or sort setting and let users change it in one click.

Use when
  • ✓ A toolbar or filter bar lets users control one named filter or sort
  • ✓ The current selection should always be visible without opening a dropdown
  • ✓ Multiple independent filters appear side-by-side in a row
  • ✓ Space is limited and a full select/dropdown would feel too heavy
Don't use when
  • ✗ The user can select multiple values at once — use a multi-select instead
  • ✗ The action triggers something immediately rather than changing a filter
  • ✗ There is no current selection to show — show a placeholder state or use a different control
  • ✗ More than ~5 Selection Buttons appear in a row — consider a dedicated filter panel

Variants

Two appearances adapt the button to different layout contexts.

Bordered — the default appearance. White background with a subtle border and shadow. Use in toolbars, filter bars, and any place the button sits on a plain background and needs to stand out as an interactive control.
Inline — no border or background in the resting state. Use inside table headers, card sections, or any dense layout where a visible border would add too much visual noise. The background appears only on hover and press to confirm interactivity.

Content guidelines

Name

The name labels the filter or dimension being controlled. Keep it to one word and end it with a colon. Use title-style capitalisation.

Do
Short, noun-based name ending in a colon.
Don't
Don't write sentences — the button becomes too wide and hard to scan.

Value

The value shows the currently selected option. Mirror the exact label used in the dropdown — don't abbreviate or rewrite it. If no option is selected, show a neutral placeholder like "All" rather than leaving the value empty.

States

The Selection Button has three interactive states. There is no disabled state — if the filter cannot be changed, hide the button or replace it with static text.

Accessibility

→
Use a native <button>. It receives keyboard focus and Enter/Space activation automatically.
→
Provide a meaningful aria-label. The visible text reads "Filter: Men" but screen readers benefit from more context. Use aria-label="Filter: Men — change filter" or link the button to a descriptive aria-describedby.
→
Announce the expanded state. If clicking opens a dropdown add aria-expanded="true/false" and aria-haspopup="listbox" so screen reader users know a list will appear.
→
Hide decorative icons. Add aria-hidden="true" to the filter icon and chevron SVG — the button text already communicates the full meaning.

Usage

Bordered (default)

<button class="selection-button">
  <eff-mat-icon [iconName]="'filter'" class="icon" />
  <span class="name">Filter:</span>
  <span class="value">Men</span>
</button>

Inline

<button class="selection-button inline">
  <eff-mat-icon [iconName]="'filter'" class="icon" />
  <span class="name">Sort:</span>
  <span class="value">Newest first</span>
</button>

With chevron

<button class="selection-button" aria-haspopup="listbox" aria-expanded="false">
  <span class="name">Status:</span>
  <span class="value">Active</span>
  <eff-mat-icon [iconName]="'chevron-down'" class="icon" />
</button>

Accessibility

→
Set aria-haspopup and aria-expanded. When the button opens a dropdown, add aria-haspopup="listbox" and toggle aria-expanded between "true" and "false" as the dropdown opens and closes.
→
Provide a clear aria-label. Combine name and value into a descriptive label, for example aria-label="Filter by gender: Men — change filter", so the button makes sense when read in isolation.
→
Mark icons as decorative. Add aria-hidden="true" to the filter and chevron SVGs so they are not announced by screen readers.
