Color

Engage uses a two-tier color system: a set of primitive color ramps that define every possible value, and a layer of semantic tokens that map those values to specific roles — surfaces, text, borders, and data. Always use a semantic token in product code. Never reference a primitive directly.

🎨 Brand palette

Four colors form the core brand identity. Turquoise is the primary action color, Navy anchors dark surfaces and all body text, Yellow is used for highlights and the product logo, and White is the default page surface.

🏷️ Semantic tokens

Semantic tokens give colors intent. Each is an alias to a primitive that can change per theme. Tokens live in three namespaces: bg/ for fills, content/ for text and icons, and border/ for strokes and dividers. Use the portal and dark-mode controls in the top bar — values update live.

✨ Best practices

Use semantic tokens — never raw primitives

Semantic tokens are theme-aware; primitives are not. Referencing color/red/100 directly in a component breaks dark mode and any future rebranding in a single line of code.

✕ This field is required
background: bg/negative/subtle
border: border/negative/subtle
color: content/negative/base
Do
Semantic tokens adapt to light and dark themes automatically.
✕ This field is required
background: #FFEBEB
border: color/red/300
color: #B9252A
Don't
Hardcoded hex values break in dark mode and require manual updates during rebrand.

Never use color alone to convey meaning

Color blindness affects roughly 8% of users. Always pair color with a label, icon, or pattern to ensure meaning is accessible to everyone.

Do
Color dot + text label — readable without any color perception.
Don't
Color-only status indicators are invisible to users with red-green color blindness.

Match the token to the semantic role

Each semantic token carries an implied meaning — negative for errors, brand for primary actions, highlight for callouts. Using a token outside its intended role confuses users who rely on color to understand context.

Do
Brand (turquoise) for primary actions, Negative (red) for destructive actions.
Don't
Swapping semantic roles causes confusion — a red "Save" button signals danger where there is none.
