Link Inline

A text link embedded inside body copy. Use it to navigate to another page, jump to a related section, or open an external resource — wherever the link reads as part of a sentence rather than a stand-alone action.

Anatomy

An inline link is just two layers — the text label and its underline. There is no container, no fill, and no padding. It inherits the font-size, line-height and weight from the surrounding paragraph so it visually belongs with the text around it.

1
Label — the link text. Inherits the font, size, weight and line-height of the parent paragraph. Keep it descriptive: link the words that explain where the link goes, never bare "click here".
2
Underline — always visible. The underline is what visually distinguishes a link from surrounding body copy when colour alone isn't enough (low-vision users, colour-blind users, screen reflections).

Usage

Use Link Inline whenever the link is part of a sentence. For stand-alone actions or in-platform navigation that sits next to other buttons, reach for a Button instead.

Use when
  • The link is embedded in a sentence and reads as part of body copy
  • You're pointing to documentation, help articles, terms, or external resources
  • Multiple links may appear close together in the same paragraph
Avoid when
  • The link is a stand-alone call-to-action — use a Button with the link variant
  • The action triggers something other than navigation (save, delete, submit)
  • The destination is unclear from the link text alone

Variants

A single inline link with three icon configurations. Pick the one that helps users predict where the link goes.

Link label
Default — text only. The most common usage; reach for this whenever the link words alone describe the destination.
Link label
Icon before — use a leading icon to flag the link's nature (e.g. a download arrow, an external-link glyph). The icon sits 6px before the label and inherits the link colour.
Link label
Icon after — use a trailing icon to indicate forward motion: chevron right for "go to", arrow up-right for "opens in a new tab". 6px gap, inherits the link colour.

Accessibility

→
Always use a native <a> element with an href. You get keyboard focus, Enter to activate, the right cursor, and proper screen-reader announcement for free.
→
Never remove the underline. Colour alone fails WCAG 1.4.1 — users with low colour vision can't tell linked text from body copy without it.
→
Link the descriptive words, not "click here". Screen readers can list every link on a page; "click here" links are useless out of context.
→
External links should announce themselves. Add a small "(opens in new tab)" hint or an icon with aria-label when the link opens in a new window.
→
Focus must be visible. The component ships a 2-px brand outline on :focus-visible — don't override it with outline:none.

CSS classes

CLASS DESCRIPTION
link-inline The base inline link. Apply to any <a> sitting inside body copy. Inherits font-size and line-height from the parent; weight is fixed at 500.
<eff-mat-icon> as first child Renders a 14×14 leading icon with a 6px gap. The icon inherits the link colour automatically.
<eff-mat-icon> as last child Renders a 14×14 trailing icon with a 6px gap. Useful for "go to" or "opens in new tab" affordances.
is-hover Forces hover styling. Use only in static documentation/state previews — never in production.
is-pressed Forces pressed styling. Use only in static documentation/state previews.
is-disabled Visually disables the link and removes pointer events. Always pair with aria-disabled="true" for screen readers.

CSS tokens

Override these tokens at the theme level to restyle every inline link at once. Don't override per-instance with inline styles.

/* Content */
--content-action:          rgba(25, 39, 67, 0.80);  /* Default       */
--content-action-hover:    rgba(25, 39, 67, 0.95);  /* Hover         */
--content-action-pressed:  rgba(25, 39, 67, 0.95);  /* Pressed       */
--content-disabled:       rgba(25, 39, 67, 0.30);  /* Disabled      */

Accessibility

→
Always use <a href>, never a <span>. A real anchor with a real href is keyboard-focusable, opens with Enter, supports right-click → "open in new tab", and is announced as a link by every screen reader.
→
External links: pair target="_blank" with rel="noopener noreferrer". noopener is a security requirement; noreferrer is good privacy hygiene.
→
Use aria-disabled="true" instead of disabled. Anchors don't support the native disabled attribute. Pair the ARIA attribute with the .is-disabled class and a click handler that prevents navigation.
→
Don't kill the focus ring. The :focus-visible outline is the only feedback keyboard users get when they tab through links — keep it.
