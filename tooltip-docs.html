Tooltip

A small, non-interactive label that appears on hover or focus to briefly explain or name a control.

Anatomy

A tooltip is a single dark bubble with a small arrow pointing at the element it describes.

Tooltip message
1
2
1
Container — A dark bubble holding one short line of text. Width grows to fit; long copy wraps.
2
Arrow — Points at the trigger and indicates the tooltip's position (above, below, left, or right).

Usage

Use a tooltip to label or briefly clarify a control — never to hide essential information. It can sit above, below, left, or right of the trigger; pick the side with the most room.

Label icon-only controls

Delete survey
Do
Name an icon-only button so its action is clear.
Save changes
Don't
Repeat a label that's already visible — it adds noise, not clarity.

Keep it to a glanceable phrase

Shown to your team only
Do
One short phrase the user can read at a glance.
This setting controls who can see your results. Choose "team" to limit visibility, or "everyone" to share across the organisation — you can change it later in settings.
Don't
Cram a paragraph in — long or essential content belongs in a Spotlight, dialog, or helper text.

Content guidelines

Tooltips are read in a glance — keep them to a short, plain phrase.

Write a short, specific phrase

One phrase that names the action or clarifies the control. No full stops needed for fragments.

Export to Excel
Do
Specific and scannable.
Click here
Don't
Vague — says nothing about the action.

Use sentence case

Capitalise the first word only; no title case and no trailing period on short fragments.

Pin to top
Do
Sentence case, no period.
Pin To Top.
Don't
Title case + period reads heavy.

Accessibility

Tooltips must be reachable by keyboard and never hide information the user needs to complete a task.

→
Show on hover and focus. The tooltip must appear both when hovering with a mouse and when the trigger receives keyboard focus — not hover only.
→
Link it to its trigger. Associate the tooltip with the control via aria-describedby so screen readers announce it.
→
Dismissable. Pressing Escape hides the tooltip without moving focus (WCAG 1.4.13).
→
Never essential-only. Don't put information the user needs to proceed only in a tooltip — it's invisible on touch and to many AT users. Use visible helper text instead.
→
No interactive content. Tooltips can't contain links or buttons. If you need actions, use a Spotlight or a dialog.
→
Contrast. White text on the inverse surface meets WCAG 2.2 AA.

Playground

Configure the tooltip and see it update — the snippet uses the Angular Material matTooltip directive.

matTooltip
matTooltipPosition

Usage

Add the matTooltip directive to any element. Set the side with matTooltipPosition.

Basic

Delete survey
<button class="tertiary" matTooltip="Delete survey" matTooltipPosition="above">
  <eff-mat-icon [iconName]="'trash'"></eff-mat-icon>
</button>

On any element

The directive works on non-button elements too (e.g. an icon or label).

<span matTooltip="Shown to your team only" matTooltipPosition="right">
  <eff-mat-icon [iconName]="'info'"></eff-mat-icon>
</span>

Inputs

The tooltip is the Angular Material matTooltip directive — there's no custom element to import.

InputTypePurpose
matTooltipstringThe text to show. Setting it also enables the directive.
matTooltipPosition'above' | 'below' | 'left' | 'right'Which side of the trigger the tooltip sits on.
matTooltipDisabledbooleanTurn the tooltip off without removing the directive.
matTooltipShowDelay · matTooltipHideDelaynumber (ms)Delay before showing / hiding.
⚠️
Figma gap Two things to reconcile. (1) Font weight — Figma uses Medium (500) for the tooltip text, but the Angular matTooltip renders it at Normal (400). (2) Rich tooltips — for a tooltip with a title, body, and actions (the "Got it" / multi-step walkthrough), use the Spotlight (eff-tooltip-dialog), not matTooltip (text-only).

Accessibility

matTooltip handles most of this; keep the content rules in mind.

→
Focus + hover. matTooltip shows on both by default and wires up aria-describedby — don't disable focus display.
→
Escape dismisses. Built in. Don't trap it.
→
Text only. Never put essential info or interactive content in a tooltip — use a Spotlight or visible helper text.
