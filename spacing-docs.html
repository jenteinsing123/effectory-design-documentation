Spacing

Engage spacing is built on a 4 px base unit. Eight named tokens cover layout spacing — from tight inline gaps to generous section breaks. Always use a token; never hardcode a pixel value.

📏 Scale

spacing-extra-tight Icon-to-label · l4–l6 to body
4 px
spacing-tight l1–l3 to body · button vertical padding
8 px
spacing-base-tight List item · table cell padding
12 px
spacing-base Default · component padding · between form fields
16 px
spacing-loose Card padding · section padding · grid gap
24 px
spacing-extra-loose Between major form groups · large cards
32 px
spacing-super-loose Between related page sections
40 px
spacing-super-extra-loose Between unrelated sections · hero padding
56 px
Beyond 56 px — the grid continues at 64, 72, 80, 88, 96, 104, 112, and 120 px for component sizing (heights, widths, fixed dimensions). These steps don't have named tokens and are referenced directly by their pixel value.

✨ Guidance

Proximity signals relationship

Items that belong together should sit closer than items that are independent. Use spacing-extra-tight between a label and its input, spacing-base between adjacent fields, and spacing-super-loose between sections that aren't directly related.

Stay on the 4 px grid

Every value in the scale is a multiple of 4. Avoid in-between values like 14, 18, or 22 px — they break visual rhythm and create misalignment with adjacent components. When something feels off, snap to the nearest grid step.

Never hardcode pixel values

Reference the named token in code, not the resolved value. When the spacing scale is adjusted, every component updates automatically — hardcoded values do not.
