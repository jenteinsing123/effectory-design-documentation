Spacing

All spacing in Engage is built on a 4 px base unit. Eight named semantic tokens cover the most common layout situations — from tight inline gaps to generous section breaks. Raw grid values (4–120 px) are available for component sizing and one-off measurements.

📏 Scale

✨ Best practices

Use named tokens — don't hardcode pixel values

Named tokens are the contract between design and code. When spacing is adjusted at the token level, every component updates automatically. Hardcoded values scatter the change across hundreds of files.

✓ Do
gap: spacing-base (16px)
Token-based spacing updates globally when the scale changes.
✕ Don't
gap: 16px   ← hardcoded
Hardcoded pixel values require manual updates every time the design scales.

Use tighter spacing for related items

Proximity signals relationship. Items that belong together should be spaced closer than items that are independent. This is the Law of Proximity — and it's the core principle behind the spacing scale.

✓ Do
4px between label and input (tightly related), 20px between fields (related but separate).
✕ Don't
Equal spacing between label and input as between fields makes labels feel unrelated to their fields.

Stay on the grid — avoid values between steps

Using values like 14px, 18px, or 22px breaks the rhythm of the 4px grid. When a layout feels off, the fix is almost always to move to the nearest grid step — not split the difference.

✓ Do
16px padding, 8px internal gap — both on the 4px grid.
✕ Don't
padding: 14px 18px; gap: 7px
Off-grid values create misalignment with adjacent elements and make layouts harder to maintain.
