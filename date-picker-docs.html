Date Picker

An interactive calendar for selecting single dates, date ranges, and optionally a time — available as a floating dropdown or embedded inline.

Anatomy

The dropdown variant consists of a trigger button that opens a floating calendar panel. The inline variant renders the panel directly without a trigger or footer.

Usage

Use a date picker whenever a structured, unambiguous date value is required — not free-form text input.

Use a picker, not a text field, for dates

A date picker guarantees a valid, correctly formatted date. Free-text fields require validation and introduce parsing errors from different locale conventions.

Start date
Do
Use a date picker to capture structured, validated date values.

Don't
Don't use a free-text field for dates — format ambiguity causes errors.

Use inline mode when the date is the main purpose of the view

The inline date picker fits naturally into scheduling flows, booking screens, and dedicated date-selection steps. A trigger + dropdown adds unnecessary friction when a date must always be chosen.

Accessibility

The date picker must be fully keyboard-navigable and announce state changes to assistive technologies.

Keyboard interaction

KeyBehaviour
Enter / SpaceOpens the picker from the trigger. Selects the focused day when the calendar is open.
EscapeCloses the dropdown and returns focus to the trigger without changing the value.
Arrow keysMove focus between day cells (left/right = prev/next day; up/down = same weekday ±1 week).
Page Up / DownNavigate to the previous or next month.
Home / EndMove focus to the first or last day of the current month.
TabMoves focus between the trigger, navigation buttons, day grid, time fields, and footer actions.

ARIA requirements

  • The calendar grid uses role="grid" with an aria-label identifying the current month and year.
  • Each day cell uses role="gridcell". Selected cells carry aria-selected="true". Disabled cells use aria-disabled="true".
  • The trigger button announces the currently selected date; if empty, it uses a descriptive placeholder such as "Select a date".
  • Month navigation buttons have explicit aria-label values ("Go to previous month", "Go to next month").
  • For date range pickers, the start and end day cells carry aria-label values that describe their role (e.g., "14 May 2026, range start").
  • When the month changes via keyboard, the new month and year are announced live via an aria-live region.

Accessibility

The component handles ARIA roles, live regions, and keyboard navigation automatically. Use the inputs below to supplement the built-in behaviour.

Required: provide a meaningful label

Always associate the date picker with a visible <label> or use aria-label / aria-labelledby to describe what date is being collected. The trigger button announces the current value; ensure the label clarifies the purpose.

<!-- With a visible label -->
<label for="startDate">Start date</label>
<eng-date-picker
  id="startDate"
  [(ngModel)]="startDate"
></eng-date-picker>

<!-- With aria-label (no visible label) -->
<eng-date-picker
  aria-label="Select start date"
  [(ngModel)]="startDate"
></eng-date-picker>

Date range: label both ends

For range pickers, use separate aria-label values or wrap in a <fieldset> + <legend> so assistive technologies can distinguish the start and end fields.

<fieldset>
  <legend>Report period</legend>
  <eng-date-picker
    [(ngModel)]="reportRange"
    [dateRange]="true"
    [footerButtons]="true"
  ></eng-date-picker>
</fieldset>
