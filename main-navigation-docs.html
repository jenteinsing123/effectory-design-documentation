Main Navigation

The primary sidebar that frames the whole platform — it switches portals, links to the top-level areas, and holds the user's account menu. One per screen, always on the left.

Anatomy

A fixed-width column with a portal switcher on top, the navigation items in the middle, and the account menu pinned to the bottom.

Content guidelines

Labels are the wayfinding system — keep them short, concrete, and stable.

Short, noun-based labels

One or two words that name the destination — not a sentence or an action.

Surveys Organization
Do
Clear nouns: "Surveys", "Organization".
Manage your surveys Org. stuff
Don't
Verbose phrases or vague abbreviations.

Accessibility

The Main Navigation is a landmark — it must be reachable and understandable by keyboard and screen readers.

→
Use the nav landmark. Wrap it in <nav aria-label="Main"> so screen-reader users can jump straight to it.
→
Mark the current page. The active item carries aria-current="page" — colour alone is not enough.
→
Expose group state. Expandable groups use aria-expanded on the header and group their sub-items so the relationship is announced.
→
Full keyboard order. Portal switcher → items → groups → Help & learn → account menu, all reachable with Tab; Enter/Space activate.
→
Icons are decorative. The label carries the meaning, so item icons are hidden from assistive tech (aria-hidden).
→
Account menu is a menu. The button uses aria-haspopup="menu" + aria-expanded; Escape closes it and returns focus to the button.
