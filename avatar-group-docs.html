Avatar Group

A row of overlapping avatars showing who's involved in something — collaborators on a survey, members of a team, attendees of a session. An overflow counter at the end caps the number of visible faces.

Anatomy

Three to five overlapping avatars followed — optionally — by a counter showing how many more aren't displayed.

Avatar stack — overlapping avatars sharing one size. Each carries a 2-px page-coloured border so the overlap is readable on any surface.

Overflow counter — a grey +N avatar shown only when the total exceeds the visible limit. Shares the size and border of the stack.

Use a group when the identities matter; use a number alone (e.g. "12 members") when they don't.

Accessibility

→
Announce as a group, not a list. Wrap the stack in role="group" with aria-label="3 of 7 collaborators" so screen readers communicate both the visible count and the total.
→
Individual avatars stay decorative. Each child avatar's tooltip still carries the name for sighted hover, but the group label already announces the overall count.
→
The counter is informational. The +N avatar's tooltip should say "3 more" so it's clear what the number represents.
→
Interactive groups need a real button. If clicking the group opens a list of all members, wrap it in a <button> with a clear label — not just a click handler on the <div>.

Usage

Place <eff-avatar> elements inside a flex container with negative left-margins to create the overlap. When the total exceeds the visible limit, append a counter avatar with text="+N".

Three avatars + counter

JI JR MY +4 
<div class="avg" role="group" aria-label="3 of 7 collaborators">
  <eff-avatar [avatar]="{ text: 'JI', tooltip: 'Jamie Insing' }" [enableDynamicColor]="true" />
  <eff-avatar [avatar]="{ text: 'JR', tooltip: 'Jordan Rivera' }" [enableDynamicColor]="true" />
  <eff-avatar [avatar]="{ text: 'MY', tooltip: 'Maya Young' }" [enableDynamicColor]="true" />
  <eff-avatar [avatar]="{ text: '+4', tooltip: '4 more' }" />
</div>
