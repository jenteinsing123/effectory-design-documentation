Announcement

A floating card that announces a new feature or update. It stays out of the way — the page underneath remains usable — and the user dismisses it when they're done.

Anatomy

A light card with an illustration, a title, a body, and a footer holding a low-emphasis dismiss action and a primary call-to-action.

We just launched 20+ additional integrations!

Managing employee data has never been easier. Discover how to connect your tools.

1
Illustration — A visual hook at the top of the card. Optional, but it's what sets this apart from a plain notification.
2
Title — A short, bold headline that names the new thing.
3
Body — One or two sentences explaining the value and what to do next.
4
Dismiss — A low-emphasis link action so the user can put it away.
5
Call-to-action — A primary button that takes the user to the feature.

Usage

Use an announcement to surface something new without interrupting. The page stays usable and it waits until the user is ready.

Content guidelines

Lead with the benefit, keep it to a sentence, and make the button say where it goes.

Say what's new and why it matters

The title names the feature; the body gives the reason to care in one short sentence.

Export your results

Do
Clear headline, one benefit-led sentence.

Update

We've shipped a number of changes across the product that we think you're going to love, so go ahead and take a look around whenever you get the chance.

Don't
Vague title and a long, unfocused body.

Make the button name the destination

Accessibility

An announcement is non-modal — it must never trap the user or steal focus.

→
Don't steal focus. Unlike a dialog or spotlight, it appears without moving focus, so the user can keep working uninterrupted.
→
Announce politely. Place it in a live region with aria-live="polite" (or a labelled role="region") so screen-reader users hear it without being interrupted.
→
Always dismissible. Provide a clearly labelled dismiss action; both buttons are reachable by keyboard in a logical tab order.
→
No auto-dismiss. It stays until the user acts, so nobody misses it or has to read against a timer.
→
The illustration is decorative. It carries no information, so hide it from assistive tech (empty alt / aria-hidden).

Usage

Toggle the dialog from a signal and anchor it to a target element. The with-svg class plus [svgUrl] gives it the illustration.

We just launched 20+ integrations!

Managing employee data has never been easier.

<!-- trigger -->
<button #svgTarget class="secondary" (click)="showAnnouncement.set(true)">
  Show announcement
</button>

<!-- announcement (with illustration) -->
<eff-tooltip-dialog
  *ngIf="showAnnouncement()"
  class="with-svg"
  [targetElement]="svgTarget"
  [svgUrl]="'assets/img/home-illustration.svg'"
  [isNewFeature]="true"
  [dialogTitle]="'We just launched 20+ integrations!'"
  [dialogSubtitle]="'Managing employee data has never been easier.'"
  [dialogButtonText]="'View integrations'"
  [dialogLinkButtonText]="'Close'"
  [dialogPositionToTarget]="'after'"
  (dialogClosedOutput)="showAnnouncement.set(false)">
</eff-tooltip-dialog>
