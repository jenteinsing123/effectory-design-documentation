Announcement

A small floating card that announces a new feature or update. It stays out of the way — the page underneath remains usable — and the user dismisses it when they're done.

Anatomy

A compact card with a full-bleed illustration, a title, a body, and two stacked full-width actions.

1
Illustration — A full-bleed image at the top. Optional, but it's what sets this apart from a plain notification.
2
Title — A short, bold headline that names the new thing.

Body — One or two sentences explaining the value.
4
Call-to-action — A full-width primary button that takes the user to the feature.

Dismiss — A small icon button in the top-right corner, over the illustration, that puts the card away.

Use an announcement to surface something new without interrupting. The page stays usable and it waits until the user is ready.

Colour

The card comes in a small set of colours drawn from the design-system palette tokens. Default is a white card with the brand (teal) button; the others tint the card background (--bg-*-subtle), the illustration (--bg-*-base) and the call-to-action. They're purely decorative colour choices — an announcement carries no error or status meaning, so pick whichever fits the moment.

Accessibility

An announcement is non-modal — it must never trap the user or steal focus.

→
Don't steal focus. Unlike a dialog or spotlight, it appears without moving focus, so the user can keep working uninterrupted.
→
Announce politely. Place it in a live region with aria-live="polite" (or a labelled role="region") so screen-reader users hear it without being interrupted.
→
Label the dismiss. The close control is an icon-only button, so give it an accessible name ("Close") and make sure it's keyboard-reachable and large enough to hit comfortably.
→
No auto-dismiss. It stays until the user acts, so nobody misses it or has to read against a timer.
→
The illustration is decorative. It carries no information, so hide it from assistive tech (empty alt / aria-hidden).

Usage

Toggle the dialog from a signal and anchor it to a target element. The with-svg class plus [svgUrl] gives it the illustration.

⚠️
Figma gap — dismiss is now an icon button The design dismisses the card with a small secondary icon button in the top-right corner of the illustration (shown in the preview). The current <eff-tooltip-dialog> only exposes a text dismiss link via [dialogLinkButtonText] (kept in the snippet below). Confirm with engineering whether to add a top-right icon-close to the component before relying on this in code.

  Show announcement

<!-- announcement (with illustration) -->
<eff-tooltip-dialog
  *ngIf="showAnnouncement()"
  class="with-svg"
  [targetElement]="svgTarget"
  [svgUrl]="'assets/img/home-illustration.svg'"
  [dialogTitle]="'Dynamic reporting is here!'"
  [dialogSubtitle]="'A new look, new features, and more to explore.'"
  [dialogButtonText]="'Learn more'"
  [dialogLinkButtonText]="'Close'"
  [dialogPositionToTarget]="'after'"
  (dialogClosedOutput)="showAnnouncement.set(false)">
</eff-tooltip-dialog>

⚠️
Stacking is app-orchestrated The deck (queueing announcements from different teams and revealing the next on close) is your app's job: keep an ordered list and show one <eff-tooltip-dialog> at a time, advancing the index when it closes. →
Decorative illustration is hidden from assistive tech (aria-hidden / empty alt).
