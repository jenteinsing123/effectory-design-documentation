Spotlight

Draws attention to a new or changed part of the UI and explains it — for feature discovery and onboarding, as a single highlight or a short multi-step tour.

Anatomy

A dark card with an optional badge, a title, a body, and a footer that holds the action.

New

Choose your settings later

Click on this icon to go to your project settings and confirm them before you create a survey.

1
Badge (optional) — A sparkle + label (e.g. "New") that flags a new feature.
2
Title — A short, bold heading naming the feature or step.
3
Body — One or two sentences explaining what to do.
4
Footer — A primary button ("Got it!"), or step dots + "Next" for a multi-step flow.
5
Arrow — Points at the element the tip refers to.

Usage

Use a spotlight to teach — to point at something new and explain it. It interrupts, so use it sparingly.

Content guidelines

A clear title and a one-sentence body that tells the user what to do.

Title names the thing; body says what to do

The title is a short noun phrase; the body is a single, action-oriented sentence.

Accessibility

A spotlight is an interactive overlay, so it needs proper focus handling — unlike a plain tooltip.

→
Move focus into it. When it opens, move focus to the card (or its action) so keyboard and screen-reader users land on it.
→
Dismissable. Escape closes it; "Got it!" / completing the tour closes it. Always offer a way out.
→
Return focus. On close, return focus to the element that triggered it (or the element it pointed at).
→
Announce as a dialog. Use role="dialog" with an aria-labelledby pointing at the title.
→
Don't block essential tasks. The user must be able to dismiss and continue without completing the tour.
→
Step progress is not colour-only. Convey "step 2 of 4" in text for screen readers, not just the dots.
