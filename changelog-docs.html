Changelog

A record of additions, changes, and fixes to the Engage Design System.

v1.4.0 — May 2025

New form components — Checkbox and Slider — alongside File Upload, and a harmonised button size token set.

New

  • Checkbox — all variants (default, bordered, ghost) across all states (unchecked, checked, indeterminate, disabled). Full documentation with anatomy, usage guidelines, and Angular API.
  • Checkbox button ghost + bordered — two additional Checkbox button variants for use in filter bars and selection lists.
  • Slider — single-thumb and range variants with step support, min/max labels, and value tooltip. Accessible via keyboard arrow keys.
  • File Upload — drag-and-drop zone with file type validation, upload progress state, and error handling for rejected files.

Changed

  • Button sizes harmonised — small, medium, and large Button sizes now use the new --size-* token set, replacing the ad-hoc height values from v1.1. No visual change for default (medium) buttons; small and large now align exactly with the spacing scale.

Fixed

  • Toggle — disabled state thumb colour was using --content-disabled instead of --bg-disabled, causing the thumb to be invisible on the disabled track in light mode.
  • Select — dropdown overlay was rendered with a stacking context below modals (z-index conflict). The dropdown now correctly appears above eff-dialog when Select is used inside a dialog.

v1.3.0 — March 2025

Four new components for progress, structure, and empty states, plus Avatar Group overflow improvements.

New

  • Progress Bar — linear progress indicator with determinate and indeterminate modes, label, and optional percentage display. Used in survey completion flows and bulk action feedback.
  • Skeleton — loading placeholder component with text, circle, and rectangle variants. Replaces ad-hoc spinner usage in content-heavy pages.
  • Empty State — structured layout for zero-content screens: illustration slot, heading, body text, and optional primary + secondary action. Covers first-use, no-results, and error-state scenarios.
  • Section Header — standard heading + optional description + optional action button layout for use above content sections within a page.

Changed

  • Avatar Group — the overflow indicator now supports up to 99+ as a label (previously capped at +9). The overflow count renders as a Badge-style pill aligned to the Avatar size.

Fixed

  • Tooltip — rapid mouse enter/leave events were causing the tooltip to flicker visible/hidden because the show and hide animations were interrupting each other. Debounce logic has been added to the show trigger.
  • Badge — positive and warning colour variants did not meet WCAG AA contrast on the --bg-subtle surface used inside Cards. Token values updated; no API change.

v1.2.0 — January 2025

Overlay and data display additions, plus an important Dialog accessibility change.

New

  • Drawer — slide-in panel anchored to the right edge, with configurable width, backdrop, and close button. Suitable for detail views and secondary workflows that don't warrant a full page navigation.
  • Context Menu — right-click / kebab-triggered action list with icon support, dividers, and a destructive item variant.
  • Popover — positioned overlay for rich non-modal content (forms, filter panels, previews). Uses the --sh-popovers shadow token and supports arrow positioning.
  • Metric — display component for a single KPI number with label, trend indicator (up/down/neutral), and optional comparison value. Used on dashboard and report summary screens.

Changed

  • Dialog — dialogs that confirm destructive or irreversible actions now render with role="alertdialog" instead of role="dialog", which causes screen readers to announce the dialog immediately on open. The [destructive] input boolean controls this automatically.

Fixed

  • Tabs — the active tab underline indicator was rendering 1px below the correct baseline in Safari due to a subpixel rounding issue on the containing flex row. Fixed by switching to an absolutely positioned pseudo-element.
  • Breadcrumb — long breadcrumb trails were overflowing their container rather than truncating the intermediate items with an ellipsis. The overflow behaviour now correctly collapses intermediate items and shows a … expander.

v1.1.0 — November 2024

Initial public release — Foundations and the full core component set.

New

  • Foundations — colour tokens (semantic + raw), typography scale (Poppins, 6 levels), spacing scale (4 px base), shadow levels, border radius scale, motion tokens (duration + easing), iconography set, and the 12-column grid system. All values exposed as CSS custom properties in tokens.css.
  • Button — primary, secondary, tertiary, danger, danger-secondary, danger-tertiary, link, and link-delete variants across default, hover, pressed, focused, and disabled states.
  • Text Field — with prefix/suffix slot, character count, helper text, and error state.
  • Select — single and multi-select with search, grouped options, and async loading state.
  • Radio Group — horizontal and vertical orientations, with optional descriptions per option.
  • Toggle — with label, helper text, and disabled state.
  • Alert — informational, positive, warning, and negative variants with optional dismiss and action link.
  • Toast — same four variants, auto-dismiss with configurable duration, and a persistent mode.
  • Dialog — with header, body, footer action slots, configurable width, and backdrop dismiss toggle.
  • Tooltip — four placement positions, delay-on-show, and a wide variant for longer descriptions.
  • Table — sortable columns, row selection, pagination integration, and an empty/loading state.
  • Card — with optional header, footer, and interactive (clickable) variant.
  • Badge and Tag — semantic colour variants and dismissible Tag.
  • Avatar — image, initials, and icon fallbacks across five sizes.
  • Tabs — with overflow scrolling and optional icon per tab.
  • Breadcrumb, Pagination — standard navigation components.

Changed

  • Initial release — no prior version to compare against.

Fixed

  • Initial release — no prior version to compare against.
